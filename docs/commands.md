# 명령 상세

이 문서는 Novel Writer Skills의 모든 Slash Commands를 상세히 소개합니다.

## 명령 분류

### 7단계 방법론 명령

| 명령 | 기능 | 출력 파일 | 의존성 |
|------|------|---------|--------|
| `/constitution` | 창작 헌법 작성 | `.specify/memory/constitution.md` | 없음 |
| `/specify` | 스토리 규격 정의 | `stories/[name]/specification.md` | `/constitution` |
| `/clarify` | 모호한 점 명확화 | `specification.md` 업데이트 | `/specify` |
| `/plan` | 창작 계획 수립 | `stories/[name]/creative-plan.md` | `/specify` |
| `/tasks` | 작업 목록 분해 | `stories/[name]/tasks.md` | `/plan` |
| `/write` | 챕터 작성 실행 | `stories/[name]/content/chapter-XX.md` | `/tasks` |
| `/analyze` | 품질 검증 분석 | 분석 보고서 (대화 내) | `/write` |

### 추적 및 검증 명령

| 명령 | 기능 | 출력 파일 | 빈도 |
|------|------|---------|------|
| `/track-init` | 추적 시스템 초기화 | `spec/tracking/*.json` | 최초 사용 |
| `/track` | 종합 추적 업데이트 | `spec/tracking/*.json` 업데이트 | 매 챕터 후 |
| `/plot-check` | 플롯 일관성 검사 | 분석 보고서 | 5-10챕터마다 |
| `/timeline` | 타임라인 관리 | `spec/tracking/timeline.json` | 중요 이벤트 후 |
| `/relations` | 캐릭터 관계 추적 | `spec/tracking/relationships.json` | 관계 변화 시 |
| `/world-check` | 세계관 검증 | 분석 보고서 | 새 설정 후 |

## 상세 설명

## `/constitution` - 창작 헌법 작성

### 기능

소설 창작의 핵심 원칙과 가치관을 수립하여 창작의 "헌법" 문서를 형성합니다.

### 사용 시나리오

- 프로젝트 시작 시
- 창작 방향 조정 시
- 새 협력자 합류 시

### 출력

`.specify/memory/constitution.md` 파일 생성, 포함 내용:
- 핵심 가치관
- 품질 기준
- 스타일 원칙
- 콘텐츠 규범

### 예시

```
/constitution
```

AI가 다음과 같은 질문을 안내합니다:
- "이 소설이 전달하려는 핵심 이념은?"
- "절대 타협할 수 없는 것은?"
- "당신의 서술 스타일은?"

### 모범 사례

- 15-20분 진지하게 생각하기
- 원칙은 구체적이고 검증 가능하게
- "반드시", "금지" 등 명확한 용어 사용
- 정기적으로 검토 및 업데이트 (버전 관리)

---

## `/specify` - 스토리 규격 정의

### 기능

제품 규격서처럼 스토리의 각 측면을 정의합니다.

### 사용 시나리오

- 새로운 스토리 아이디어가 있을 때
- 생각을 정리해야 할 때
- 정식 창작 시작 전

### 출력

`stories/[story-name]/specification.md` 파일 생성, 포함 내용:
- 한 줄 요약
- 타겟 독자 프로필
- 핵심 셀링 포인트
- 주요 캐릭터
- 성공 기준
- 제약 사항 및 리스크

### 예시

```
/specify
```

AI가 질문합니다:
- "이 스토리를 한 문장으로 설명해주세요"
- "타겟 독자는 누구인가요?"
- "핵심 갈등은 무엇인가요?"
- "스토리의 독특한 점은?"

### 특수 표시

규격 문서에서 사용:
- `[명확화 필요]` - 모호한 점 표시
- `[핵심 요구사항]` - 타협 불가 요구사항
- `[선택 기능]` - 금상첨화 내용

### 모범 사례

- 먼저 빠르게 초안 작성 후 세분화
- SMART 원칙으로 성공 기준 설정
- "아닌 것"과 "인 것"을 명확히
- 문서를 간결하게 유지 (3-5페이지 권장)

---

## `/clarify` - 모호한 점 명확화

### 기능

5개의 정밀한 질문으로 규격의 모호함을 제거합니다.

### 사용 시나리오

- `/specify` 완료 후
- 규격이 모호한 경우
- 팀원들이 규격에 대해 다른 이해를 가질 때

### 과정

1. AI가 `specification.md` 스캔
2. 모호하거나 모순된 부분 식별
3. 최대 5개의 핵심 질문 생성
4. 답변에 따라 규격 자동 업데이트
5. 결정 이력 보존

### 예시

```
/clarify
```

가능한 질문:
- "주인공의 핵심 동기는 복수인가요, 정의인가요?"
- "결말은 해피엔딩인가요, 여운을 남기나요?"
- "스토리 페이스는 빠른 전개인가요, 느린 전개인가요?"

### 모범 사례

- 솔직하게 답변, "정답"을 위해 답하지 말기
- 정말 모르겠으면 "생각해볼게요"라고 말하기
- 답변이 후속 창작에 영향을 주므로 신중하게
- 명확해질 때까지 여러 번 실행 가능

---

## `/plan` - 창작 계획 수립

### 기능

규격을 구체적인 기술 방안과 챕터 구성으로 변환합니다.

### 사용 시나리오

- 규격이 명확해진 후
- 글쓰기 시작 전
- 스토리 구조를 재설계할 때

### 출력

`stories/[story-name]/creative-plan.md` 파일 생성, 포함 내용:
- 글쓰기 방법 선택 (3막 구조, 영웅의 여정 등)
- 챕터 구성 설계
- 인물 관계도
- 세계관 체계
- 타임라인 및 인과 관계
- 복선 회수 계획

### 예시

```
/plan
```

AI가 도와줍니다:
- 적합한 글쓰기 방법 선택
- 챕터 아웃라인 설계
- 페이스와 긴장감 기획
- 캐릭터 아크 매핑

### 모범 사례

- 계획은 구체적이되 과도하게 상세하지 않게
- 영감을 위한 여유 공간 남기기
- 앞 1/3 내용에 중점 기획
- 정기적으로 검토 및 조정

---

## `/tasks` - 작업 목록 분해

### 기능

창작 계획을 실행 가능한 작업으로 분해합니다.

### 사용 시나리오

- 계획 완료 후
- 명확한 실행 경로가 필요할 때
- 다인 협업 시

### 출력

`stories/[story-name]/tasks.md` 파일 생성, 포함 내용:
- 챕터 작성 작업
- 캐릭터 프로필 완성 작업
- 세계관 설정 작업
- 수정 및 교정 작업

### 작업 표시

- `[P]` - 병렬 실행 가능
- `[의존:X]` - 작업 X에 의존
- `[높음]` - 높은 우선순위
- `[P0/P1/P2]` - 우선순위 표시

### 예시

```
/tasks
```

다음과 같이 생성:
```
[P0] 주인공 캐릭터 프로필 완성
[P0][의존:1] 제1장 쓰기: 프롤로그
[P1] 세계관 지리 설정 보충
[P1][P] 제2장 쓰기: 첫 만남
```

### 모범 사례

- 작업 크기 적절하게 (2-4시간 완성 가능)
- 의존 관계 표시로 병목 방지
- 크리티컬 패스 작업 우선 완료
- 정기적으로 작업 상태 업데이트

---

## `/write` - 챕터 작성 실행

### 기능

작업 목록과 계획에 기반하여 AI가 보조하는 챕터 창작.

### 사용 시나리오

- 작업 준비 완료 후
- 매 글쓰기 세션
- AI 보조 콘텐츠 생성이 필요할 때

### 출력

`stories/[story-name]/content/chapter-XX.md` 파일 생성.

### 사용 방법

```
/write

# 또는 챕터 지정
/write 제5장 - 첫 만남

# 또는 작업 기반
/write [P0][의존:1] 제1장 쓰기: 프롤로그
```

### 글쓰기 시 자동 활성화되는 Skills

- **Genre Knowledge**: 스토리 장르에 맞는 관례 적용
- **Dialogue Techniques**: 대화 장면 최적화
- **Scene Structure**: 장면 리듬 제어
- **Consistency Checker**: 백그라운드 일관성 검사

### 모범 사례

- 한 번에 1-2챕터 작성
- specification과 plan 참조
- AI가 먼저 프레임워크 생성 후 세부사항 채우기
- 여러 버전 저장하여 비교 선택

---

## `/analyze` - 품질 검증 분석

### 기능

작품 품질과 일관성을 전면 검증합니다.

### 두 가지 모드

1. **프레임워크 분석** (글쓰기 전)
   - 기획의 완전성 검증
   - 논리성 점검
   - 실행 가능성 평가

2. **콘텐츠 분석** (글쓰기 후)
   - 실행 품질 검증
   - 일관성 점검
   - 달성도 평가

### 사용 시나리오

- 처음 3챕터 완료 후 (조기 검증)
- 매 5챕터마다 (정기 점검)
- 초고 완성 후 (전면 검토)

### 검증 차원

```
/analyze
```

점검 항목:
- ✅ 헌법 준수 여부
- ✅ 규격 충족도
- ✅ 콘텐츠 일관성
- ✅ 품질 기준
- ✅ 페이스와 긴장감
- ✅ 캐릭터 아크

### 모범 사례

- 일찍 실행하여 일찍 문제 발견
- 모두 쓸 때까지 기다리지 말기
- 분석 결과에 따라 계획 조정
- 개선점 기록

## 추적 명령 상세

## `/track-init` - 추적 시스템 초기화

### 기능

추적 시스템에 필요한 JSON 파일을 생성합니다.

### 사용 시나리오

- 추적 기능 최초 사용 시
- 새 스토리 시작 시

### 출력

`spec/tracking/`에 생성:
- `plot-tracker.json` - 플롯 추적
- `timeline.json` - 타임라인
- `character-state.json` - 캐릭터 상태
- `relationships.json` - 캐릭터 관계

### 예시

```
/track-init
```

### 모범 사례

- 글쓰기 시작 전에 실행
- 한 번만 실행하면 됨
- JSON 파일 수동 편집 가능

---

## `/track` - 종합 추적 업데이트

### 기능

모든 추적 데이터를 스마트하게 업데이트합니다.

### 사용 시나리오

- 매 챕터 완료 후
- 중요한 이벤트 발생 후
- 정기 정리 시

### 과정

1. AI가 최신 챕터 읽기
2. 핵심 정보 추출
3. 모든 추적 파일 업데이트
4. 수동 확인이 필요한 항목 표시

### 예시

```
/track

# 또는 챕터 범위 지정
/track 제3-5장
```

### 모범 사례

- 매 챕터 후 즉시 실행하는 습관 기르기
- AI의 업데이트가 정확한지 확인
- AI가 놓친 세부사항 수동 보충

---

## 빠른 참조

### 최소 워크플로우

```
/constitution → /specify → /write
```

### 표준 워크플로우

```
/constitution → /specify → /clarify → /plan → /tasks → /write → /analyze
```

### 완전 워크플로우

```
/constitution → /specify → /clarify → /plan → /tasks → 
/track-init → /write → /track → /analyze
```

### 권장 일정

```
1일차: /constitution + /specify (2-3h)
2일차: /clarify + /plan (2-3h)
3일차: /tasks + /write 시작 (2-3h)
4-10일차: /write (매일 1-2h)
11일차: /analyze + 다음 배치 계획
```

## 명령 조합 기술

### 빠른 반복

```
1. /write 제1-3장
2. /analyze 조기 검증
3. 피드백에 따라 /plan 조정
4. /write 계속
```

### 심층 기획

```
1. /constitution (핵심 원칙)
2. /specify (전체 규격)
3. /clarify (모호함 제거)
4. /plan (상세 계획)
5. 완전히 명확해질 때까지 /clarify 반복
6. /write 시작
```

### 탐색적 창작

```
1. 간소화된 /constitution
2. 경량 /specify
3. 바로 /write으로 탐색적 챕터
4. 결과에 따라 /specify와 /plan 업데이트
5. 구조화된 /write
```
