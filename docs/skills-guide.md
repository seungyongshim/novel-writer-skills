# Agent Skills 사용 가이드

Agent Skills는 Novel Writer Skills의 핵심 기능입니다 - 창작 시 **자동으로 활성화**되며, 수동 호출이 필요 없습니다.

## Skills 작동 원리

### 자동 활성화

Claude Code에서 창작할 때, AI가 맥락에 따라 어떤 Skills를 활성화할지 자동으로 판단합니다:

```
당신: "로맨스 소설을 쓰고 싶어요"
     ↓
[Romance Skill 자동 활성화]
     ↓
AI 응답 시 로맨스 소설의 창작 관례를 자동 적용
```

### 지속 적용

한 번 활성화되면, Skills는 전체 대화에서 활성 상태를 유지합니다:
- 감지 없이 실행
- 지속적인 전문 조언 제공
- 문제 감지 시 능동적으로 알림

## Skills 분류

### 1. Genre Knowledge Skills (장르 지식 라이브러리)

#### Romance (로맨스 소설)

**활성화 조건**:
- "로맨스", "사랑 이야기", "연애" 언급 시
- 스토리 규격에 로맨틱 요소 포함 시

**제공 내용**:
- 감정 리듬 포인트 (첫 만남, 첫 키스, 어둠의 순간, 대단원)
- 페이스 가이드 (슬로우번 vs 패스트번)
- 흔한 함정 (첫눈에 반함, 오해만을 유일한 갈등으로 사용)
- HEA/HFN 결말 요구사항

**적용 시점**:
- `/specify` - 관계 아크 포함 제안
- `/plan` - 감정 리듬을 챕터에 매핑
- `/write` - 대화와 케미스트리 기법
- `/analyze` - 로맨스 관례 준수 확인

#### Mystery (미스터리/추리)

**활성화 조건**:
- "미스터리", "추리", "탐정", "범죄" 언급 시
- 스토리에 수수께끼와 수사 요소 포함 시

**제공 내용**:
- 페어 플레이 원칙
- 단서 배치 전략 (3단계)
- 오도 단서(레드 헤링) 설계
- 용의자 관리

**적용 시점**:
- `/specify` - 핵심 미스터리 정의
- `/plan` - 단서 배치 타임라인 작성
- `/write` - 단서가 보이지만 명확하지 않도록 보장
- `/analyze` - 페어 플레이 검증

#### Fantasy (판타지)

**활성화 조건**:
- "판타지", "마법", "세계관 구축" 언급 시
- 스토리에 가상 세계 요소 포함 시

**제공 내용**:
- 샌더슨의 마법 법칙
- 마법 시스템 설계 프레임워크
- 세계관 구축 체크리스트
- 종족 및 생물 설계 원칙

**적용 시점**:
- `/specify` - 핵심 세계 요소 정의
- `/plan` - 세계 규칙 일관성 확보
- `/write` - 행동 속에서 세계 보여주기
- `/analyze` - 세계 규칙 일관성 검사

### 2. Writing Techniques Skills (글쓰기 기법)

#### Dialogue Techniques (대화 기법)

**활성화 조건**:
- 대화 장면 작성 시
- 캐릭터 대화에 관한 질문 시

**제공 내용**:
- 언어를 통한 캐릭터 표현
- 서브텍스트가 직설보다 우선
- 끊김과 겹침 기법
- 정보 덤핑 방지

**적용 시점**:
- `/write` - 대화 장면 자동 최적화
- 실시간 캐릭터 목소리 일관성 제안
- 직설적 대화 감지 및 알림

#### Scene Structure (장면 구조)

**활성화 조건**:
- 장면 기획 또는 작성 시
- 챕터 구조 관련 질문 시

**제공 내용**:
- 장면-후속장면 모델
- 목표-갈등-재앙 프레임워크
- 긴장감 관리 전략
- 리듬 포인트 설계

**적용 시점**:
- `/plan` - 장면 시퀀스 설계
- `/write` - 장면 구축 가이드
- 장면 아웃라인 템플릿 제공

### 3. Quality Assurance Skills (품질 보증)

#### Consistency Checker (일관성 검사)

**활성화 조건**:
- 작성 과정에서 **자동**으로 백그라운드 실행
- 수동 활성화 불필요

**모니터링 내용**:
- 캐릭터 일관성 (외모, 성격, 지식)
- 세계 규칙 일관성
- 타임라인 논리성

**알림 형식**:

```
⚠️ 일관성 검사 알림

문제: 캐릭터 특성 불일치
위치: 제5장, 제3단락
참조: characters/mary.md, 15행

현재 텍스트: "메리의 초록색 눈..."
확립된 특성: "눈 색상: 파란색"

가능한 해결책:
1. 현재 텍스트를 "파란 눈"으로 변경
2. 캐릭터 프로필 업데이트 (설정 변경 시)
3. 다른 캐릭터인가요?

자동 수정할까요?
```

**심각도**:
- 🔴 **심각**: 즉시 수정
- ⚠️ **경고**: 가능한 빨리 수정
- 📝 **참고**: 확인 필요

#### Workflow Guide (워크플로우 가이드)

**활성화 조건**:
- 사용자가 "소설을 쓰고 싶어요"라고 할 때
- 새 프로젝트 시작 시
- 권장 프로세스에서 벗어날 때

**제공 내용**:
- 7단계 방법론 가이드
- 단계 의존성 설명
- 모범 사례 조언
- 일반적인 문제 처리

**부드러운 알림**:

```
직접 글쓰기로 건너뛴 것을 알아챘습니다. Novel-writer의
장점은 체계적인 사전 기획에 있습니다. /constitution과
/specify를 실행하여 탄탄한 기반을 만들까요?
```

## Skills와 Commands 협업

### 완전한 워크플로우 예시

```
사용자: "로맨스 미스터리 소설을 쓰고 싶어요"

[자동 활성화]
✓ Romance Skill
✓ Mystery Skill  
✓ Workflow Guide Skill

AI: "좋습니다! 체계적인 방법으로 창작합시다. 먼저
/constitution을 실행하여 창작 원칙을 정의하세요..."

사용자 실행: /constitution
[Workflow Guide가 안내 제공]

사용자 실행: /specify
[Romance + Mystery Skills가 장르 맞춤 제안 제공]

사용자 실행: /write
[활성화된 모든 Skills가 협업]
- Romance: 감정 리듬 포인트
- Mystery: 단서 배치
- Dialogue Techniques: 대화 최적화
- Consistency Checker: 백그라운드 모니터링

[문제 감지]
⚠️ 일관성 알림: 캐릭터가 제3장에서 모르던 정보를
제5장에서 갑자기 알고 있음
```

## Skills 설정

### 엄격도 조정

일관성 검사의 엄격도를 조정할 수 있습니다:

```
"판타지 소설이라 마법이 현실을 왜곡할 수 있으니
유연 모드로 일관성 검사를 해주세요."
```

**모드**:
- **엄격 모드**: 모든 모순 표시
- **유연 모드**: "멋진 규칙" 예외 허용
- **최소 모드**: 핵심 모순만 표시

### 특정 검사 비활성화

의도적인 불일치인 경우:

```
"꿈 시퀀스에 대해 타임라인 검사를 비활성화해주세요 -
의도적으로 비선형입니다."
```

## Skills 활성화 테스트

특정 Skill이 활성화되었는지 확인하려면 직접 물어보세요:

```
"현재 어떤 Skills가 활성화되어 있나요?"
```

AI가 현재 활성 Skills 목록을 알려줍니다.

## 자주 묻는 질문

### Q: Skills가 창작에 너무 간섭하지 않나요?

A: 아닙니다! Skills의 설계 원칙:
- **제안하되, 강제하지 않음**: 전문적 조언을 제공하지만 최종 결정권은 당신에게
- **수동 활성화**: 창작 흐름을 방해하지 않음
- **설정 가능**: 조정하거나 비활성화 가능

### Q: 수동으로 특정 Skill을 활성화할 수 있나요?

A: 필요 없습니다! 관련 내용을 언급하기만 하면 됩니다:
- Romance Skill 활성화? "이건 로맨스 소설이에요"라고 말하세요
- Mystery Skill 활성화? "여기에 수수께끼가 있어요"라고 말하세요

### Q: 여러 Skills가 동시에 활성화되면 충돌하나요?

A: 아닙니다! Skills는 협업하도록 설계되었습니다:
- Romance + Mystery = 로맨틱 미스터리
- Fantasy + Romance = 판타지 로맨스
- 모든 Skills는 서로의 존재를 알고 있습니다

### Q: Skills가 응답 시간을 늘리나요?

A: 거의 없습니다! Skills는 경량입니다:
- 대부분 백그라운드에서 실행
- 관련 있을 때만 제안 제공
- 일반 대화 속도에 영향 없음

### Q: 제안이 어떤 Skill에서 온 것인지 어떻게 알 수 있나요?

A: 보통 알 필요가 없습니다! 하지만 궁금하다면:
- 제안에 출처가 포함됩니다 (예: "로맨스 관례에 따르면...")
- "이 제안은 어떤 Skill에서 온 건가요?"라고 물어볼 수 있습니다

## Skills와 플러그인의 차이

| 특성 | Agent Skills | 플러그인 |
|------|-------------|---------|
| **활성화 방식** | AI 자동 판단 | 사용자 설치 |
| **위치** | `.claude/skills/` | `plugins/` |
| **유형** | 지식 라이브러리, 검사 시스템 | 명령, 전문가, 기능 |
| **인지도** | 무인지 | 명시적 (Slash 명령) |
| **용도** | 수동 보조 | 능동 기능 |

**간단히 말해서**:
- Skills = 자동 활성화되는 전문 지식
- 플러그인 = 수동 호출하는 도구와 기능

## 커스텀 Skills 개발

직접 Skills를 만들고 싶으신가요? [플러그인 개발 가이드](plugin-development.md)를 참조하세요.

Skills의 기본 구조:

```yaml
---
name: my-custom-skill
description: "Use when [트리거 조건] - [기능 설명]"
allowed-tools: Read, Grep
---

# Skill Title

## Quick Reference
[빠른 참조]

## Core Principles
[핵심 원칙]

## Integration with Commands
[명령과의 통합]
```

## 요약

Agent Skills는 당신의 **보이지 않는 조수**입니다:
- ✨ 자동 활성화, 수동 조작 불필요
- 🎯 정확한 전문 지식 적용
- 🔍 능동적 문제 발견
- 🤝 명령과 협업
- 🎛️ 설정 및 조정 가능

**당신은 창작에 집중하고, Skills가 품질을 책임집니다!**
