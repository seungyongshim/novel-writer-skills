창작 계획을 기반으로 구체적이고 실행 가능한 작업 목록을 생성합니다.

## 목표

거시적 계획을 미시적 작업으로 전환하여, 창작을 관리 가능하고 추적 가능하게 만듭니다.

## 실행 단계

### 1. 계획 문서 로드

`{SCRIPT}`를 실행하여 로드:
- 창작 계획: `stories/*/creative-plan.md`
- 챕터 구조 정보
- 타임라인 및 의존 관계

### 2. 작업 목록 생성

`stories/*/tasks.md`를 생성하며, 다음을 포함:

#### 핵심 집필 작업

**중요**: specification.md 제5장과 creative-plan.md의 단서 분포에서, 각 집필 작업에 관련 단서를 표기합니다.

```markdown
## 집필 작업

### 고우선순위 [반드시 먼저 완료]

- [ ] [P0] **T001** - 제1장: [챕터 제목] (목표 글자수)
  - **핵심 작업**: [본 챕터의 주요 작업]
  - **핵심 플롯**: [구체적 플롯 포인트]
  - **관련 단서**:
    - PL-XX([단서 이름]) ⭐⭐⭐ 주진행
    - PL-YY([단서 이름]) ⭐ 배경
  - **교차점**: [해당 시] X-001([교차점 설명])
  - **복선 배치/해소**: [해당 시] F-001 배치 / F-002 해소
  - **반드시 포함**: [핵심 요소 목록]
  - **장말 훅**: [서스펜스 설정]
  - **의존**: 없음
  - **출력**: `content/volume1/chapter-001.md`

- [ ] [P0] **T002** - 캐릭터 프로필: 주인공 상세 설정
  - **핵심 작업**: 주인공 설정 완성
  - **반드시 포함**: 성격, 배경, 능력, 욕망, 공포, 성장 아크
  - **의존**: 없음
  - **출력**: `characters/protagonist.md`

### 중우선순위 [일반 진행]

- [ ] [P1] **T005** - 제5장: [챕터 제목] (목표 글자수)
  - **핵심 작업**: [본 챕터 주요 작업]
  - **핵심 플롯**: [구체적 플롯]
  - **관련 단서**:
    - PL-01([단서명]) ⭐⭐⭐ 주진행
    - PL-02([단서명]) ⭐⭐ 보조
  - **교차점**: 없음
  - **복선 배치/해소**: F-001 배치([복선 내용])
  - **반드시 포함**: [핵심 요소]
  - **장말 훅**: [서스펜스]
  - **의존**: T004(제4장)
  - **출력**: `content/volume1/chapter-005.md`

### 저우선순위 [선택적 보완]

- [ ] [P2] 번외: 캐릭터 프리퀄
- [ ] [P2] 설정집: 상세 세계관
```

**작업 필드 설명**:
- **관련 단서**: creative-plan.md의 "활성 단서" 열에서 읽어, 본 챕터에서 추진하는 단서를 표기
- **교차점**: specification.md 5.3절에서 읽어, 본 챕터가 교차점인지 표기
- **복선 배치/해소**: specification.md 5.4절에서 읽어, 본 챕터에서 관련된 복선 작업 표기

#### 작업 표기 설명
- `[P]` - 병렬 실행 가능
- `[의존:X]` - 작업 X를 먼저 완료해야 함
- `[P0/P1/P2]` - 우선순위 표기

### 3. 작업 정렬 및 그룹핑

- 우선순위별 그룹핑
- 의존 관계 식별
- 병렬 가능 작업 표기
- 완료 시간 추정

### 4. 실행 계획 생성

```markdown
## 실행 계획

### 1단계 (1주차)
병렬 작업 그룹1:
- 주인공 설정 [P]
- 세계관 기초 [P]
- 제1장 초고 [P]

### 2단계 (2-3주차)
순차 작업:
- 제2장 [의존:제1장]
- 제3장 [의존:제2장]

병렬 작업 그룹2:
- 조연 설정 [P]
- 장면 설계 [P]
```

### 5. 작업 통계 출력

- 총 작업 수
- 예상 총 글자수
- 예상 완료 시간
- 핵심 마일스톤

다음 단계 안내: `/write`를 실행하여 집필을 시작하거나 특정 작업 상세 내용을 확인하세요.
