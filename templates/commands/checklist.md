# 품질 검사 체크리스트 (Checklist)

품질 검사 체크리스트를 생성하거나 실행하며, 두 가지 모드를 지원합니다:

## 🎯 지원하는 검사 유형

### 제1유형: 사양 품질 검사 (질문 생성식)
기획 문서 자체의 품질을 검증합니다 ("요구사항의 단위 테스트"와 유사):

- `대강 품질` - outline.md의 완전성, 명확도, 일관성 검사
- `캐릭터 설정` - spec/knowledge/characters.md 검사
- `세계관` - spec/knowledge/world-setting.md 및 관련 문서 검사
- `창작 계획` - creative-plan.md / specification.md 검사
- `복선 관리` - spec/tracking/plot-tracker.json의 복선 정의 검사

### 제2유형: 내용 검증 검사 (결과 보고식)
작성된 챕터를 스캔하여, 실제 내용을 검증합니다:

- `세계관 일관성` - 챕터 내용을 스캔하여 세계관 묘사 모순 검사
- `플롯 정렬` - 진도와 대강을 비교하여 플롯 발전 검사
- `데이터 동기화` - 모든 tracking JSON 파일의 동기화 검증
- `타임라인` - 시간 이벤트의 논리 연속성 검사
- `집필 상태` - 집필 준비도와 작업 상태 검사

## 사용자 입력

```text
$ARGUMENTS
```

## 실행 흐름

### 1. 검사 유형 식별

사용자 입력에 따라 검사 유형을 결정합니다 (사양 품질 vs 내용 검증):

**키워드 매핑**:
- "대강", "품질" → 사양 품질류: 대강 품질
- "캐릭터", "설정" → 사양 품질류: 캐릭터 설정
- "세계관" + "품질/완전성/사양" → 사양 품질류: 세계관
- "세계관" + "일관성/검사/스캔" → 내용 검증류: 세계관 일관성
- "창작 계획", "기획" → 사양 품질류: 창작 계획
- "복선" → 사양 품질류: 복선 관리
- "플롯", "정렬", "진도" → 내용 검증류: 플롯 정렬
- "데이터", "동기화", "일관성" → 내용 검증류: 데이터 동기화
- "타임라인", "시간" → 내용 검증류: 타임라인
- "집필 상태", "준비" → 내용 검증류: 집필 상태

사용자 입력이 불명확하면 선택을 질문합니다.

### 2. 해당 검사 로직 실행

#### 사양 품질류 검사 (질문 생성식 Checklist)

spec-kit과 유사한 요구사항 품질 검증 로직을 실행합니다:

##### 2.1 대강 품질 검사

**목표**: outline.md가 우수한 완전성, 명확도, 일관성을 갖추었는지 검증합니다.

**읽을 파일**:
- `outline.md` 또는 `stories/*/outline.md`
- `spec/tracking/plot-tracker.json` (존재하는 경우)

**검사 항목 차원 생성**:

**완전성 (Completeness)**:
- 모든 주요 플롯 노드에 트리거 조건과 결과가 정의되었는가?
- 각 권/장의 스토리 목표가 명확한가?
- 모든 주요 캐릭터의 성장 아크를 커버하는가?
- 주요 갈등의 에스컬레이션 경로가 정의되었는가?
- 스토리의 클라이맥스와 결말이 명확한가?

**명확도 (Clarity)**:
- 플롯 노드의 트리거 조건이 구체적으로 검증 가능한가?
- 챕터 배분에 명확한 근거가 있는가 (예: 글자수, 플롯 밀도)?
- 캐릭터 동기가 구체적 사건으로 정량화되었는가?
- 장면 묘사에서 모호한 용어를 피했는가 ("어떤 곳", "한동안")?

**일관성 (Consistency)**:
- 플롯 단서가 앞뒤로 모순되지 않는가?
- 캐릭터 행동이 설정에 부합하는가?
- 시간 간격이 합리적인가?
- 세계관 규칙이 대강 묘사와 일치하는가?

**측정 가능성 (Measurability)**:
- 챕터 배분이 합리적으로 실현 가능한가 (예: 챕터당 2000-4000자)?
- 복선 해소 시기가 명확한가 (챕터 번호 또는 범위)?
- 캐릭터 성장에 명확한 마일스톤이 있는가?

**커버리지 (Coverage)**:
- 모든 주요 장면 유형을 고려했는가 (갈등, 일상, 전환점)?
- 모든 주요 캐릭터의 출연 분량을 커버했는가?
- 필요한 복선 배치와 해소가 포함되어 있는가?

**출력 형식 예시**:
```markdown
# 대강 품질 검사 체크리스트
**생성 시간**: 2025-10-11
**검사 대상**: outline.md
**검사 차원**: 완전성, 명확도, 일관성, 측정 가능성, 커버리지

## 완전성 (Completeness)

- [ ] CHK001 모든 주요 플롯 노드에 트리거 조건과 결과가 정의되었는가? [Spec §대강3.2]
- [ ] CHK002 각 권/장의 스토리 목표가 명확한가? [Gap]
- [ ] CHK003 모든 주요 캐릭터의 성장 아크를 커버하는가? [Spec §대강5.1]

## 명확도 (Clarity)

- [ ] CHK004 플롯 노드의 트리거 조건이 구체적으로 검증 가능한가? [Ambiguity, Spec §대강3.2]
- [ ] CHK005 챕터 배분에 명확한 근거가 있는가? [Clarity]

## 일관성 (Consistency)

- [ ] CHK006 플롯 단서가 앞뒤로 모순되지 않는가? [Consistency]
- [ ] CHK007 세계관 규칙이 대강 묘사와 일치하는가? [Consistency, vs §세계관]

## 측정 가능성 (Measurability)

- [ ] CHK008 챕터 배분이 합리적으로 실현 가능한가 (예: 챕터당 2000-4000자)? [Measurability]
- [ ] CHK009 복선 해소 시기가 명확한가 (챕터 번호 또는 범위)? [Gap]

## 커버리지 (Coverage)

- [ ] CHK010 모든 주요 장면 유형을 고려했는가? [Coverage]
- [ ] CHK011 필요한 복선 배치와 해소가 포함되어 있는가? [Coverage, Gap]

## 사용 설명

검증 완료 항목 체크: `[x]`
문제 항목 표기: `[!]` 아래에 구체적 문제 기록
```

##### 2.2 캐릭터 설정 검사

**읽을 파일**:
- `spec/knowledge/characters.md`
- `spec/tracking/character-state.json`
- `spec/tracking/relationships.json`

**검사 항목 차원 생성**:

**완전성**:
- 주요 캐릭터에 기본 정보가 정의되었는가 (이름, 나이, 신분, 외모)?
- 캐릭터의 핵심 동기와 목표가 정의되었는가?
- 캐릭터의 성격 특성과 행동 패턴이 정의되었는가?
- 캐릭터의 배경 스토리가 정의되었는가?
- 캐릭터의 능력과 한계가 정의되었는가?

**명확도**:
- 캐릭터 동기가 구체적으로 검증 가능한가 ("성공하고 싶다"가 아닌 "과거를 통해 가문의 운명을 바꾸고 싶다")?
- 성격 특성이 구체적 행동으로 체현되는가?
- 캐릭터 목표가 정량화 가능하거나 명확한 달성 기준이 있는가?

**일관성**:
- 캐릭터 설정이 대강의 행동과 일치하는가?
- 다른 문서의 캐릭터 묘사가 일치하는가?
- 캐릭터 관계 정의가 대칭적인가 (A→B 관계 vs B→A 관계)?

**측정 가능성**:
- 캐릭터 성장에 명확한 단계 구분이 있는가?
- 캐릭터 능력 변화가 추적 가능한가?

##### 2.3 세계관 검사

**읽을 파일**:
- `spec/knowledge/world-setting.md`
- `spec/knowledge/locations.md`
- `spec/knowledge/culture.md`
- `spec/knowledge/rules.md`

**검사 항목 차원 생성**:

**완전성**:
- 핵심 세계관 규칙이 정의되었는가 (마법 체계, 기술 수준, 사회 구조)?
- 주요 장소와 그 특징이 정의되었는가?
- 문화 풍속, 언어, 전통이 정의되었는가?
- 시대 배경과 역사 맥락이 정의되었는가?

**명확도**:
- 세계관 규칙이 명확하고 모호하지 않은가?
- 지리 위치, 거리, 방위가 명확한가?
- 특수 용어에 명확한 정의가 있는가?

**일관성**:
- 다른 문서의 세계관 설정이 일치하는가?
- 세계관 규칙에 내부 모순이 있는가?
- 대강 묘사와 일치하는가?

**커버리지**:
- 스토리에 등장하는 모든 장소를 커버했는가?
- 등장하는 모든 특수 규칙이나 능력이 정의되었는가?

##### 2.4 창작 계획 검사

**읽을 파일**:
- `creative-plan.md` 또는 `specification.md`
- `tasks.md`

**검사 항목 차원 생성**:

**완전성**:
- 창작 목표와 마일스톤이 정의되었는가?
- 창작 프로세스와 단계가 명확한가?
- 품질 기준이 정의되었는가?

**명확도**:
- 작업 분할이 명확하고 구체적인가?
- 시간 배치가 합리적인가?
- 인수 기준이 명확한가?

**일관성**:
- 계획이 대강 규모와 매칭되는가?
- 작업이 모든 기획 내용을 포괄하는가?

##### 2.5 복선 관리 검사

**읽을 파일**:
- `spec/tracking/plot-tracker.json`
- `outline.md`

**검사 항목 차원 생성**:

**완전성**:
- 모든 계획된 복선이 기록되었는가?
- 각 복선에 배치 챕터와 해소 챕터가 정의되었는가?
- 복선의 유형과 중요도가 정의되었는가?

**명확도**:
- 복선 내용 설명이 명확한가?
- 해소 방식이 명확한가?

**측정 가능성**:
- 해소 시기에 명확한 챕터 번호 또는 범위가 있는가?
- 배치 밀도가 정의되었는가 (과다 미해소 복선 방지)?

**일관성**:
- 복선이 대강 플롯과 매칭되는가?
- planted와 resolved 필드가 일치하는가?

### 3. Checklist 출력

**저장 위치**: `spec/checklists/`

**파일 명명 규칙**:
- 사양 품질류: `[유형]-quality.md` (예: `outline-quality.md`)
- 내용 검증류: `[유형]-[날짜].md` (예: `world-consistency-20251011.md`)

**출력 형식**: `templates/checklist-template.md`를 템플릿으로 사용합니다.

### 4. 결과 보고

출력:
- Checklist 파일 경로
- 검사 항목 총수
- 검사 유형과 범위
- Checklist 사용 방법 설명

## 사용 예시

```bash
# 사양 품질 검사
/checklist 대강 품질
/checklist 캐릭터 설정
/checklist 세계관

# 내용 검증 검사
/checklist 세계관 일관성
/checklist 플롯 정렬
/checklist 데이터 동기화
```

## 주의사항

1. **사양 품질류 checklist**: 집필 전 기획 검증에 사용, 문서 자체의 품질 문제 발견
2. **내용 검증류 checklist**: 집필 후 내용 검사에 사용, 실제 산출물의 문제 발견
3. 두 유형의 checklist은 상호 보완적이며, 권장: 기획 단계에서 제1유형, 집필 단계에서 제2유형 사용
4. 모든 checklist은 `spec/checklists/` 디렉토리에 저장, 검사 이력 추적 용이

## 하위 호환성 설명

이전 명령 `/world-check`와 `/plot-check`는 여전히 사용 가능하지만, 통합된 `/checklist` 명령 사용을 권장합니다.
