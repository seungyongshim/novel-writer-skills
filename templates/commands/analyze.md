소설 프로젝트에 대해 스마트 종합 분석을 수행합니다. 현재 창작 단계에 따라 **프레임워크 일관성 분석** (write 전) 또는 **콘텐츠 품질 분석** (write 후)을 자동 선택합니다.
---

## 핵심 이념

**하나의 명령어, 이중 스마트**:
- 📐 **프레임워크 분석**: 글쓰기 전 스펙, 계획, 작업의 일관성 검증 (spec-kit 유사)
- 📝 **콘텐츠 분석**: 글쓰기 후 완성된 콘텐츠의 품질과 부합도 검증

**절제되지만 단순하지 않음**:
- 사용자는 `/analyze`만 실행하면 시스템이 어떤 분석을 실행할지 자동 판단
- 수동 모드 지정 지원: `$ARGUMENTS --type=framework` 또는 `--type=content`

## 실행 프로세스

### 1. 스마트 단계 감지

`{SCRIPT}`를 실행하여 현재 창작 상태를 가져옵니다:

```json
{
  "analyze_type": "framework|content",
  "chapter_count": 0,
  "has_spec": true,
  "has_plan": true,
  "has_tasks": true,
  "story_dir": "/path/to/story",
  "reason": "사유 설명"
}
```

### 2. 결정 로직

사용자 파라미터 `$ARGUMENTS` 파싱:

**수동 지정 모드** (최우선순위):
- `--type=framework` 포함 → 강제 프레임워크 분석
- `--type=content` 포함 → 강제 콘텐츠 분석

**🆕 전문 분석 모드** (신규):
- `--focus=opening` 포함 → 오프닝 전문 분석 (처음 3챕터 중점 분석)
- `--focus=pacing` 포함 → 리듬 전문 분석 (쾌감 포인트/갈등 분포 중점 분석)
- `--focus=character` 포함 → 인물 전문 분석 (인물 아크 중점 분석)
- `--focus=foreshadow` 포함 → 복선 전문 분석 (복선 매설/회수 중점 분석)
- `--focus=logic` 포함 → 논리 전문 분석 (논리 허점 중점 탐색)
- `--focus=style` 포함 → 스타일 전문 분석 (문체 일관성 중점 검사)

**자동 판단 모드**:
- 챕터 수 = 0 → **프레임워크 분석**
- 챕터 수 < 3 → **프레임워크 분석** (글쓰기 계속 가능 안내)
- 챕터 수 ≥ 3 → **콘텐츠 분석**

### 3. 해당 분석 실행

결정 결과에 따라 다음 두 가지 분석 중 하나를 실행합니다.

## 모드 A: 프레임워크 일관성 분석

**목표**: 글쓰기 전 준비 작업이 충분한지 검증하고, 스펙, 계획, 작업 간 모순이 없는지 확인합니다.

### A1. 기준 문서 로드

- 헌법 파일: `.specify/memory/constitution.md`
- 스펙 파일: `stories/*/specification.md`
- 계획 파일: `stories/*/creative-plan.md`
- 작업 파일: `stories/*/tasks.md`

### A2. 커버리지 분석

모든 스펙 요구사항에 대응하는 계획과 작업이 있는지 확인:

```markdown
## 커버리지 분석 보고서

### P0 요구사항 커버리지
- [요구사항1: 주인공 성장선] → ✅ 계획 제3장 구간, 작업#5-8
- [요구사항2: 빌런 설정] → ⚠️ 계획에 언급되었으나 구체적 작업 없음
- [요구사항3: 서스펜스 설정] → ❌ 계획과 작업 모두에서 미커버

### P1 요구사항 커버리지
커버리지율: 75% (3/4)

### P2 요구사항 커버리지
커버리지율: 50% (2/4)

### 작업 완전성
- 모든 계획 챕터에 대응 작업이 있는가: ⚠️ 제10-12장 작업 부족
- 작업이 모든 핵심 장면을 포함하는가: ✅ 예
```

### A3. 일관성 검사

문서 간 모순이 존재하는지 검증:

```markdown
## 일관성 검사 보고서

### 스펙 ↔ 계획
- ✅ 주제 표현 일관
- ⚠️ 스펙은 "빠른 리듬" 요구하나, 계획 앞 5챕터 리듬이 느림
- ❌ 스펙은 "감정선 과다" 금지하나, 계획 제6-8장에 대량 감정선

### 계획 ↔ 작업
- ✅ 모든 계획 챕터에 작업 있음
- ⚠️ 작업 총 글자수 예상 150K이나, 계획 목표는 100K
- ❌ 계획은 제5장을 클라이맥스로 요구하나, 작업은 "전환 챕터"로 표기

### 헌법 준수
- ✅ 계획이 창작 헌법의 가치관에 부합
- ✅ 작업 분해가 품질 기준에 부합
```

### A4. 논리 문제 경고

스토리 라인 설계의 잠재적 논리 허점 분석:

```markdown
## 논리 문제 경고

### 타임라인 충돌
- ⚠️ 제3장은 "3년 후"이나, 제5장에서 캐릭터가 "2년 전의 일"을 언급, 시간이 맞지 않음

### 캐릭터 능력 모순
- ❌ 제2장 주인공 "무공을 모름", 제4장 작업 설명 "검술로 적을 격파"

### 복선 미계획
- ⚠️ 제1장에 "신비한 패"라는 복선을 깔았으나, 후속 챕터에 회수 계획 없음
```

### A5. 준비 완료 평가

글쓰기를 시작할 수 있는지 평가:

```markdown
## 준비 완료 평가

### 필수 조건 (P0)
- [x] 스펙이 완전하고 명확
- [x] 계획이 모든 P0 요구사항을 커버
- [ ] 작업 분해 완전 (3개 챕터 작업 부족)
- [ ] 치명적 논리 모순 없음 (2곳 발견)

### 권장 조건 (P1)
- [x] 캐릭터 프로필 완비
- [ ] 세계관 설정 문서 불충분
- [x] 타임라인 계획 명확

### 종합 점수: 6/10

**제안**:
1. 🔴 반드시 수정: 제10-12장 작업 보충
2. 🔴 반드시 수정: 타임라인과 캐릭터 능력 모순 해결
3. 🟡 최적화 권장: 세계관 설정 문서 보충
4. 🟢 선택: 앞 5챕터 리듬 설계 조정

**결론**: 현재 **글쓰기 시작을 권장하지 않음**, 먼저 P0 문제를 해결하세요.
```

### 🆕 B5.1 전문 분석 (선택)

**사용자가 `--focus` 파라미터를 지정한 경우, 해당 전문 심층 분석을 실행합니다**:

#### 전문1: 오프닝 분석 (--focus=opening)

**목표**: 처음 1-3챕터가 황금 오프닝 법칙에 부합하는지 심층 분석

**분석 차원**:

```markdown
## 오프닝 전문 분석 보고서

### 황금 법칙 검사

**`spec/presets/golden-opening.md`가 존재하면 자동으로 읽어 5대 법칙을 적용합니다**

#### 법칙1: 동적 장면 진입
- ✅ 제1장 오프닝 방식: [액션/대화/갈등] 직접 진입
- ❌ 문제 발견: 오프닝에 200자 정적 환경 묘사 (법칙 위반)
- 제안: 삭제하거나 50자 이내로 축소, 바로 액션으로 진입

#### 법칙2: 핵심 갈등 전치
- ✅ 핵심 갈등 제시 시점: 제1장 제[X]절
- ⚠️ 갈등 강도: 보통 ("주인공 생존/목표 위협" 수준으로 격상 제안)
- 구체: [갈등 내용 설명]

#### 법칙3: 정보 폭격 회피
- ✅ 세계관 공개 방식: 점적식, 플롯에 자연스럽게 융합

#### 법칙4: 등장인물 수 제한
- ✅ 제1장 이름 있는 캐릭터: [X]명 (≤3명 요구사항 충족)
- ❌ 문제 발견: 제1장 등장 5명, 과다 (법칙 위반)
- 제안: [캐릭터D]와 [캐릭터E]의 등장을 제2-3장으로 지연

#### 법칙5: 골든 핑거 신속 전시
- ✅ 골든 핑거 전시 시점: 제[X]장
- ⚠️ 전시 방식: 언급만 있고, 실제 사용 없음 (실제 효과 시연 제안)
- 구체: [전시 방식 설명]
```

## 주의사항

### 스마트하지만 제어 가능
- 자동 모드가 90%의 사용 시나리오 커버
- 수동 모드로 특수 요구 대응
- 사용자가 복잡한 규칙을 외울 필요 없음

### 🆕 전문 분석의 사용 시나리오

**언제 전문 분석을 사용하는가?**

1. **--focus=opening**: 처음 3챕터 완료 후 즉시 사용
   - 오프닝은 독자 유지의 핵심
   - 황금 오프닝 법칙은 강제 요구사항
   - 조기 발견이 비용이 더 낮음

2. **--focus=pacing**: 10-15챕터마다 한 번 사용
   - 리듬이 예상에 부합하는지 확인
   - 쾌감 포인트/갈등 분포 합리성 평가
   - rhythm-config에 따라 리듬 조정

3. **--focus=character**: 주요 전환점 이후 사용
   - 주인공이 중대한 사건을 겪은 후
   - 조연 등장 또는 퇴장 시
   - 인물 관계가 변화할 때

4. **--focus=foreshadow**: 매 권 완성 후 사용
   - 복선 누락 여부 확인
   - 복선 매설 품질 평가
   - 회수 시점 사전 계획

5. **--focus=logic**: 대강 조정 후 사용
   - 중요한 설정 수정 후
   - 타임라인 조정 후
   - 챕터 내용 추가/삭제 후

6. **--focus=style**: 일괄 수정 전 사용
   - style-reference와 비교하여 일관성 확인
   - AI 어투와 금지 단어 발견
   - 스타일이 벤치마크 작품에 부합하는지 확인

**전문 분석과 전체 분석의 관계**:
- **전체 분석** (기본): 단계별 검사에 적합 (5-10챕터마다)
- **전문 분석** (--focus): 타겟 최적화에 적합 (문제 발견 시)

**권장 워크플로**:
1. 5-10챕터 완료마다 → `/analyze` (전체 분석)
2. 오프닝 문제 발견 → `/analyze --focus=opening`
3. 리듬이 안 맞는 느낌 → `/analyze --focus=pacing`
4. 논리가 불확실 → `/analyze --focus=logic`
5. 수정 전 확인 → `/analyze --focus=style`

### 객관적이고 건설적
- 데이터와 기준에 기반한 분석
- 주관적 판단 회피
- 구체적이고 실행 가능한 제안 제공

### 점진적 개선
- 분석은 개선을 위한 것이지, 비판이 아님
- 매번 분석 결과 기록
- 개선 효과 추적

### 🆕 다른 기능과의 시너지

**전문 분석이 자동 읽는 파일**:
- `spec/presets/golden-opening.md` → opening 분석
- `spec/presets/rhythm-config.json` → pacing 분석
- `memory/style-reference.md` → style 분석
- `stories/*/specification.md` → 모든 분석의 기준

**장점**:
- 참조 파일을 수동 지정할 필요 없음
- 벤치마크 작품 기준 자동 적용
- 분석 기준 일관성 유지
