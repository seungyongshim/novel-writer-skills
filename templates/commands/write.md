## 글쓰기 실행 프로세스

### 1. 글쓰기 작업 선택
`tasks.md`에서 상태가 `pending`인 글쓰기 작업을 선택하고, `in_progress`로 표시합니다.

### 2. 전제 조건 검증
- 관련 의존 작업이 완료되었는지 확인
- 필요한 설정이 준비되었는지 검증
- 이전 챕터가 완료되었는지 확인

### 3. 글쓰기 전 알림
**헌법 원칙 기반 알림**:
- 핵심 가치관 요점
- 품질 기준 요구사항
- 스타일 일관성 준칙

**스펙 요구사항 기반 알림**:
- P0 반드시 포함할 요소
- 타겟 독자 특성
- 콘텐츠 레드라인 알림

**글쓰기 스타일 및 규범 기반 알림 (설정된 경우)**:
- 현재 활성화된 글쓰기 스타일 및 핵심 원칙
- 현재 활성화된 글쓰기 규범 및 핵심 요구사항
- 스타일과 규범의 조합 효과 설명
- 특별히 주의할 금기사항 및 요점

**예시**:
```
🎨 현재 글쓰기 설정:
- 스타일: natural-voice (자연스러운 목소리)
  - 구어체 우선, 대화로 플롯 추진
  - 행동 > 심리, 구체적 > 추상적

- 규범: anti-ai-v4 + fast-paced
  - 200+ 금지 단어, 형용사 제한
  - 챕터당 최소 2개 쾌감 포인트, 빠른 리듬

조합 효과: 자연스럽고 유창한 빠른 리듬의 통쾌한 글
```

**단락 형식 규범 (중요)**:
- ⛔ **금지**: "하나", "둘", "셋" 등 숫자 표기로 단락 구분
- ✅ **사용 방식**: 장면 전환 시 두 줄 빈 줄 (한 개의 빈 행)로 구분
- 📖 **이유**: 숫자 표기는 너무 딱딱하여 읽기 몰입감을 해치며, 웹소설 관례에 맞지 않음

**안티 AI 감지 글쓰기 규범 (텐센트 주작 기준 기반)**:

⚠️ **중요 배경**: AI 프로그래밍 도구는 낮은 온도 파라미터를 사용하지만, 전통적 "보상 방법" (강제 세부사항 쌓기)은 과도한 묘사를 초래하여 오히려 AI 특성을 증가시킵니다. 다음 규범은 실측 통과 기준 (AI 농도 0%)을 기반으로 합니다.

### 📏 단락 구조 규범 (핵심) ⭐

**단문 단락 비율**:
- ✅ **30%-50%의 단락이 단문 단락이어야 함**
- ✅ **각 단락 50-100자 이내로 제어**
- ✅ **핵심 정보는 독립 단락으로**

**예시 비교**:

❌ **AI식 글쓰기** (과도한 묘사, 95% AI 농도):
> 방에는 곰팡이 냄새가 가득했고, 유일한 광원은 커튼 틈새로 스며드는 회백색 달빛이었다. 그는 벽을 더듬으며 전진했고, 손끝이 차가운 석벽에 닿았으며, 무릎이 탁자 모서리에 부딪혔다 — 흔들거리는 낡은 나무 탁자에는 먼지가 잔뜩 쌓여 있었다.

✅ **자연스러운 글쓰기** (간결하고 절제, 0% AI 농도):
> 영가의 난 이후, 중원은 이민족에게 점령당했다.
>
> 한족 사족과 백성들은 고향을 떠나기 싫은 소수를 제외하고 대부분 남쪽으로 강을 건넜다.
>
> 왕초는 이 몇 년 동안 백여 명의 유민을 모아 자기 땅을 경작하게 했다.

### 🚫 금지 사항 목록 (안티 AI 어투)

1. **무의미한 나열 금지**
   - ❌ 억지로 "3가지 감각"을 채우지 말 것
   - ❌ 나열식 감정 묘사 금지
   - ✅ 정확한 세부사항 하나가 나열된 세 개보다 나음

2. **화려한 비유 금지**
   - ❌ "흔들거리는 낡은 탁자", "공기가 얼어붙다"
   - ✅ 직접 묘사: "낡은 나무 탁자", "침묵"

3. **과도한 드라마화 금지**
   - ❌ "말이 끝나기도 전에, 그녀는 이미 돌아서 떠나고 있었다. 그가 달려가 붙잡으며..."
   - ✅ 간결하게: "그녀가 돌아섰다. 그가 뒤쫓았다."

4. **설명식 대사 금지**
   - ❌ "나는 화가 나, 네가 어제 안 왔으니까"
   - ✅ "어제 어디 갔어?" "……너한테 상관없어."

5. **직접적 심리 묘사 금지**
   - ❌ "그는 속으로 생각했다, 이 일은 간단하지 않다고"
   - ✅ 행동으로 암시: "그의 눈썹이 찌푸려졌다."

### ✅ 자연스러운 글쓰기 원칙

**1. 역사 백묘법** (고대 배경에 적합)
- 사실을 진술, 수식 없이
- 예시: "이 몇 년 동안 왕초는 백여 명의 유민을 모아 자기 땅을 경작하게 했다."

**2. 구어체 처리** (대화)
- 말실수, 멈춤, 반복 추가
- 예시: "대부분 사람들이 남하했지" (정확하지 않은 표현 허용)

**3. 짧은 문장 리듬** (서사)
- 단문 15-25자
- 핵심 정보 독립 단락

**4. 절제된 묘사** (장면)
- 한 장면에 1-2개 세부사항이면 충분
- ❌ 쓰지 말 것: "방에는 곰팡이 냄새가 가득하고, 벽은 차갑고, 빛은 어두웠다..."
- ✅ 대신: "방은 어두웠다." (충분함)

### 📊 자가 점검 기준

한 단락을 쓴 후 점검:
- [ ] 단문 단락 비율이 30%-50%인가?
- [ ] 각 단락 글자수가 50-100자인가?
- [ ] "유일한", "결국", "가득하다" 등 AI 고빈도 단어가 있는가?
- [ ] 감각 세부사항을 억지로 쌓았는가?
- [ ] 대화가 너무 완전한가 (멈춤, 말실수 부족)?
- [ ] 비유가 너무 화려한가?

**AI 고빈도 단어 블랙리스트**:
- "유일한", "결국", "가득하다", "흔들거리는"
- "공기가 얼어붙다", "말이 끝나기도 전에", "갑자기"
- "자신도 모르게", "순간", "속으로 생각하다"
- "미간을 찌푸리다", "한숨을 쉬다"

**대체 전략**:
| ❌ AI 어휘 | ✅ 자연스러운 대체 |
|---------|----------|
| 곰팡이 냄새가 가득하다 | 곰팡이 냄새가 난다 |
| 유일한 광원 | 빛이 조금 있을 뿐 |
| 흔들거리는 낡은 탁자 | 낡은 나무 탁자 |
| 속으로 생각하다 | 생각하다 / 삭제 |
| 말이 끝나기도 전에 | 말을 다 못 했는데 / 삭제 |

### 4. 실시간 보조 모드 (선택)

**사용자가 글쓰기 중 어려움을 겪을 때**, 예를 들어:
- "주인공이 어떻게 해야 할지 생각 좀 도와줘"
- "다음에 플롯을 어떻게 전개하지?"
- "선택지 몇 개 줘"

**2-3개 행동 선택지를 능동적으로 제공할 수 있습니다**, 예를 들어:

> **플롯 전개 제안**:
>
> **선택지 A (능동형)**: 주인공이 직접 나서서 치트키로 상대를 압도
> - 장점: 직접적 쾌감, 독자 만족감 강함
> - 리스크: 주인공이 너무 강해 보일 수 있음
>
> **선택지 B (전략형)**: 주인공이 실력을 숨기고 지략으로 승리
> - 장점: 주인공의 지혜를 보여주고 긴장감 추가
> - 리스크: 리듬이 다소 느려질 수 있음
>
> **선택지 C (반전형)**: 새로운 변수 도입으로 현재 갈등 중단
> - 장점: 복잡성 추가, 새로운 단서 도출
> - 리스크: 독자가 중단감을 느낄 수 있음

**사용자의 선택에 따라** 창작을 계속합니다.

⚠️ **주의**: 이것은 보조 모드입니다. 사용자가 명시적으로 도움을 요청하지 않는 한 선택지를 능동적으로 제공하지 마세요.

- 글자수 요구사항에 도달했는가

**형식 규범 검사**:
- ⚠️ "하나", "둘", "셋" 등 숫자 표기 단락 구분을 사용하지 않았는지 확인
- ✅ 장면 전환에 두 줄 빈 줄 (한 개의 빈 행) 사용
- ✅ 단락 간격이 자연스럽고 유창한지 유지

### 📊 구체화 점검 목록 (AI 어투 제거 핵심) ⭐

한 단락을 쓴 후, 추상적 표현을 능동적으로 식별하고 대체합니다:

#### 🔍 추상적 표현 식별

**시간 추상** ❌ → **구체화** ✅
- "최근" → "지난 수요일 오후"
- "아주 오래전" → "3년 전 가을"
- "얼마 전" → "어제 아침 8시"
- "한참 지나서" → "꼬박 두 시간을 기다린 후"

**인물 추상** ❌ → **구체화** ✅
- "많은 사람들" → "내 주변에 최소 5명의 친구가"
- "누군가 말하길" → "이 아저씨가 나에게" / "옆집 할머니가 말하길"
- "다들 알다시피" → "마을 어른들이 모두 말하길"
- "들리는 바에 의하면" → "왕 아저씨가 사석에서 말한 바로는"

**수량 추상** ❌ → **구체화** ✅
- "효과가 좋다" → "이번에 지난번보다 곡식 세 섬을 더 수확" / "손님이 평소보다 두 배"
- "매우 비싸다" → "한 끼에 30만 원이 들었다"
- "매우 멀다" → "차로 두 시간 걸린다"
- "매우 많다" → "최소 스무 개는 된다"

**장면 추상** ❌ → **구체화** ✅
- "방이 어지럽다" → "바닥에 사흘째 안 빤 옷이 쌓여 있다"
- "날씨가 춥다" → "내쉬는 숨이 하얗게 보인다"
- "매우 피곤하다" → "산길을 꼬박 다섯 시간 걸었다"
- "분위기가 긴장된다" → "아무도 말이 없고, 시계 째깍 소리만 들린다"

#### 💡 능동적 검색 제안

**다음 상황을 만나면, WebSearch로 실제 세부사항을 얻는 것을 고려하세요**:
- 역사적 사건: 실제 날짜, 인물, 장소 검색
- 기술적 세부사항: 실제 파라미터, 전문 용어 검색
- 지리 정보: 실제 지명, 거리, 랜드마크 검색
- 문화 풍습: 현지 방언, 풍습, 특산물 검색
- 데이터 뒷받침: 실제 통계, 사례, 뉴스 검색

**검색 공식**:
```
- "한국 [시대] 관직 체계"
- "[도시명] 특색 방언 어휘"
- "[연대] 실제 역사적 사건"
- "[업종] 전문 용어 대전"
```

#### ✅ 구체화 자가 점검 질문

- [ ] 시간이 구체적인가? ("최근", "오래전" 금지)
- [ ] 인물 출처가 명확한가? ("누군가", "다들" 금지)
- [ ] 수량이 정확한가? ("매우 많다", "적지 않다" 금지)
- [ ] 장면 세부사항이 눈에 보이는가? ("매우 ~하다"식 형용 금지)
- [ ] 실제 지명/인명/데이터를 사용했는가?
- [ ] 대화에 구체적 내용이 있는가? ("그는 많은 말을 했다" 금지)

#### 📌 구체화 주의사항

**적도 원칙**:
- ✅ 핵심 플롯은 반드시 구체화: 전환점, 클라이맥스, 복선
- ✅ 중요한 세부사항은 반드시 구체화: 첫인상, 핵심 소도구
- ⚠️ 부차적 정보는 개괄 가능: 전환 단락, 배경 깔기
- ❌ 과도한 구체화 금지: 유수장 식 나열, 장황함

**장면 적응**:
- 고대 배경: 역사 백묘법, 적도 구체화
- 현대 배경: 생활 세부사항, 높은 구체화
- 판타지 배경: 세계관 설정, 적도 구체화

**예시 비교**:

❌ **추상 버전** (AI 어투):
```
최근 도시에서 많은 일이 있었고, 다들 이야기하고 있었다. 왕강은 듣고 나서 매우 걱정이 되어, 가서 상황을 살펴보기로 했다.
```

✅ **구체화 버전** (현실감):
```
지난 수요일부터, 시장의 이 아줌마가 계속 동쪽 거리에서 사고가 났다고 떠들었다.

왕강은 이틀을 듣다가 참을 수 없었다: "도대체 무슨 일이야?"

"사람이 죽었어!" 이 아줌마가 목소리를 낮추며, "마트 하던 장 아저씨래..."

왕강 마음이 덜컥했다. 장 아저씨는 아는 사이다, 지난달에 거기서 쌀을 샀는데.

오후에 가보기로 했다.
```

**구체화 효과 비교**:
- 시간: 최근 → 지난 수요일
- 장소: 도시 → 동쪽 거리, 시장
- 인물: 다들 → 이 아줌마, 장 아저씨
- 사건: 많은 일 → 사람이 죽다, 마트 운영
- 세부사항: 들리는 바 → 목소리를 낮추며, 지난달 쌀 구매

### 8. 글자수 검증 및 진행 업데이트

**글자수 통계 설명**:
- 정확한 한국어 글자수 통계 방법 사용
- Markdown 표기 (`#`, `*`, `-` 등) 제외
- 실제 콘텐츠 글자만 통계
- 글자수 요구사항은 `spec/tracking/validation-rules.json`에서 옴 (기본 2000-4000자)

**검증 방법**:
프로젝트 제공 글자수 통계 스크립트로 챕터 글자수 검증:
```bash
source scripts/bash/common.sh
count_chinese_words "stories/*/content/제X장.md"
```

⚠️ **주의**: `wc -w`로 한국어 글자수를 세지 마세요, 매우 부정확합니다!

**완료 보고서**:
```
✅ 챕터 집필 완료

- 실제 글자수: [X]자
- 글자수 요구사항: 2000-4000자
- 글자수 상태: ✅ 요구사항 충족 / ⚠️ 글자수 부족 / ⚠️ 글자수 초과
- 작업 상태: 업데이트 완료
```
