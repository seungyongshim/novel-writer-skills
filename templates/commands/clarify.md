## 목표

스토리 대강의 모호성이나 누락된 결정 사항을 감지하고, 대화형 질의응답으로 명확화 정보를 수집하며, 결과를 스토리 파일에 기록합니다.

**주의**: 이 명확화 프로세스는 `/plan` 이전에 실행하고 완료해야 합니다. 사용자가 명확화를 명시적으로 건너뛰겠다고 하면 (예: 탐색적 창작) 계속할 수 있지만, 후속 재작업 위험이 증가한다고 경고해야 합니다.

## 실행 단계

### 1. 초기화 확인

`{SCRIPT}`를 실행하여 현재 스토리 경로 가져오기:
- JSON을 파싱하여 `STORY_PATH`와 `STORY_NAME` 가져오기
- 스토리 파일을 찾지 못한 경우, 먼저 `/story`를 실행하여 스토리 대강 생성을 안내
- 스토리 파일 내용을 로드하여 분석

### 2. 구조화된 모호성 스캔

스토리 대강을 전면 스캔하여, 각 카테고리의 명확도를 평가합니다 (명확/부분 명확/누락):

**창작 포지셔닝**
- 타겟 독자층 (연령대, 성별 경향, 독서 수준)
- 작품 포지셔닝 (상업 웹소설/순수문학/장르소설)
- 예상 규모 (단편 3-5만자/중편 10-20만자/장편 50만자+)

**세계관 설정**
- 시대 배경 정확도 (구체적 연도/조대/가공 정도)
- 세계 규칙 (마법 체계/기술 수준/사회 제도)
- 지리 범위 (단일 도시/다국/대륙/우주)

**캐릭터 설계**
- 주인공 성장 곡선 (약자 역전/천재형/차근차근)
- 주인공 성격 기조 (열혈/냉정/복흑/성녀)
- 조연 기능 포지셔닝 (줄거리 추진/감정 지지/대비 반영)
- 빌런 지능 설정 (바보 빌런/호각지세/고차원 압도)

**서사 전략**
- 시점 선택 (1인칭/3인칭 제한/전지적 시점)
- 타임라인 구조 (선형 서사/역순 삽입/다선 병행)
- 서사 리듬 (빠른 템포 웹소설/느린 전개/완급 조절)

**플롯 핵심**
- 핵심 갈등 유형 (인간vs인간/인간vs자연/인간vs사회/인간vs자아)
- 주선 목표 명확도 (복수/성장/구원/탐험)
- 결말 경향 (해피엔딩/비극/오픈엔딩)

**스타일 기조**
- 문체 선택 (구어체 유창/고풍 우아/유머 해학/냉정 사실주의)
- 묘사 중점 (액션 장면/심리 묘사/환경 분위기/대화 추진)
- 감정 기조 (열혈 격앙/억압 암울/따뜻한 힐링/가슴 아픈 감동)

**창작 제약**
- 민감 내용 처리 (폭력 수위/감정 척도)
- 가치관 지향 (긍정 에너지/현실주의/비판적)
- 업데이트 계획 (매일/매주/매월)

각 "부분 명확" 또는 "누락" 카테고리에 대해 후보 질문을 생성합니다, 다음 경우를 제외:
- 명확화가 창작 방향에 실질적으로 영향을 미치지 않는 경우
- 챕터 계획 단계에서 결정하는 것이 더 적합한 정보인 경우

### 3. 우선순위 질문 대기열 생성

최대 5개의 우선순위 명확화 질문을 내부적으로 생성하며, 다음 제약을 적용:
- 전체 세션에서 최대 5개 질문
- 각 질문은 다음 중 하나로 답변 가능해야 함:
  * 객관식 (2-5개 상호 배타적 선택지)
  * 단답형 (5단어 이내로 제한)
- 창작 방향에 실질적 영향이 있는 질문만 포함
- 카테고리 커버리지 균형 보장, 영향력이 높은 영역 우선
- 5개 이상의 카테고리가 명확화 필요 시, (영향력×불확실성) 최고인 5개 선택

### 3.5 질문 설계 원칙 (대화식 이해)

각 질문은 **실제 작가가 대화하듯** 이루어져야 하며, 엔지니어가 양식을 채우듯이 하면 안 됩니다.

**핵심 원칙**:
- ✅ **먼저 관찰을 말하고, 그다음 질문**: 작가가 "왜 이것을 묻는지" 이해하게 함
- ✅ **자연스러운 언어 사용**: 창작자의 언어로, 분류 학술 용어가 아닌
- ✅ **영향 지적**: 이 결정이 무엇에 영향을 미치는지 작가가 이해하게 함
- ❌ **직접 질문 던지기 지양**: 처음부터 "당신의 타겟 독자는?" 하고 묻지 않기

**비교 예시**:

❌ **공학적 질문** (지양):
```
질문1: 타겟 독자의 연령대는?
A. 18-25세  B. 26-35세  C. 36-45세
```

✅ **대화식 질문** (권장):
```
💬 스토리에 학원 요소와 직장 내용이 함께 있는 것을 알았습니다. 이 두 장면의 타겟 독자층은 상당히 다릅니다——
학원 독자는 열혈 성장을 좋아하고, 직장인 독자는 권모술수에 더 관심이 많습니다. 이것은 리듬 설계와 가치관 표현에 직접 영향을 미칩니다.

그래서 먼저 확인하고 싶습니다: **주로 누구를 위해 쓰시나요?**

| 선택지 | 설명 |
|--------|------|
| A | 학생 (18-25세) - 성장과 이상주의 중심 |
| B | 직장인 (26-35세) - 현실과 전략 사고 중심 |
| C | 양쪽 모두 (이중 서사로 조정, 둘 다 충족) |
| D | 직접 입력 (당신의 생각을 입력해주세요) |
```

**질문 구조 템플릿**:
```markdown
💬 [관찰된 현상/모순점]. [이것이 어떤 창작 결정에 영향을 미치는가].

확인하고 싶은 것: **[핵심 질문]**

[선택지 표 또는 단답 안내]
```

### 4. 순차 문답 루프

**한 번에 하나의 질문만 표시**, 대화식 형식으로 질문합니다.

객관식 형식 (반드시 질문 배경 포함):
```markdown
💬 [질문 배경 설명: 무엇을 관찰했는가? 무엇에 영향을 미치는가?]

확인하고 싶은 것: **[핵심 질문]**

| 선택지 | 설명 |
|--------|------|
| A | 선택지A 상세 설명 |
| B | 선택지B 상세 설명 |
| C | 선택지C 상세 설명 |
| D | 선택지D 상세 설명 |
| E | 선택지E 상세 설명 (선택사항) |
| F | 직접 입력 (당신의 생각을 입력해주세요) |
```

단답형 형식 (반드시 질문 배경 포함):
```markdown
💬 [질문 배경 설명: 무엇을 관찰했는가? 무엇에 영향을 미치는가?]

확인하고 싶은 것: **[핵심 질문]**

간단히 답해주세요 (5단어 이내): _______
```

**사용자 답변 처리**
- 답변 유효성 검증
- F (직접 입력) 선택 시, 사용자의 자유 입력 수신 및 기록
- 모호한 경우, 빠른 명확화 요청
- 답변 기록 후 다음 질문으로 진행

**중지 조건**
- 모든 핵심 모호성이 해결됨
- 사용자가 완료를 표시함 ("됐어요", "충분해요", "그만할게요")
- 5개 질문 상한에 도달

### 5. 명확화 결과 통합

각 수락된 답변 후 즉시:

**최초 통합 시**
- 스토리 대강에 `## 명확화 기록` 섹션 생성 (존재하지 않는 경우)
- `### 명확화 세션 [날짜]` 부제목 추가

**기록 형식**
```markdown
- 질문: [질문 내용] → 답변: [사용자 답변]
```

**관련 섹션 업데이트**
명확화 내용에 따라 스토리 대강의 해당 부분 업데이트:
- 창작 포지셔닝 → 스토리 개요 업데이트
- 세계관 → 세계관 설정 업데이트
- 캐릭터 → 캐릭터 설정 업데이트
- 서사 전략 → 창작 설명에 추가
- 스타일 기조 → 스타일 가이드에 추가

### 6. 검증 및 저장

매 업데이트 후 검증:
- 명확화 기록 완전성
- 새 답변으로 해결된 모호 표기가 남아있지 않은지
- 모순되는 서술이 없는지
- Markdown 형식 정확성

업데이트된 내용을 스토리 파일에 다시 기록합니다.

## 행동 규칙

- 의미 있는 모호성이 발견되지 않으면: "즉시 명확화가 필요한 핵심 모호성이 감지되지 않았습니다."로 응답
- 스토리 파일이 누락된 경우, 먼저 `/story` 실행을 안내
- 총 5개 질문 제한을 초과하지 않음
- 순수 기술적 집필 세부사항 질문 회피
- 사용자의 조기 종료 신호 존중
- 질문 없이 완전 커버리지 달성 시, 간결한 커버리지 요약 출력

## 소설 창작 특정 고려사항

- **장르 적응**: 스토리 장르 (웹소설/미스터리/로맨스/순수문학 등)에 따라 해당 지식 베이스 로드, 타겟 질문 제공
- **독자 지향**: 상업 작품 vs 문학 작품의 다른 기준과 중점
- **문화 민감성**: 특정 소재는 특별히 신중한 처리 필요
- **시리즈 계획**: 시리즈 작품 여부가 전체 구조와 결정에 영향
