# 종합 진도 추적

소설 창작의 각종 진도와 상태를 전면적으로 표시합니다.

## 추적 차원

1. **집필 진도** - 글자수, 챕터, 완료율
2. **플롯 발전** - 주선 진도, 부선 상태
3. **타임라인** - 스토리 시간 진행
4. **캐릭터 상태** - 캐릭터 발전 및 위치
5. **복선 관리** - 배치 및 해소 상태

## 사용 방법

스크립트 {SCRIPT} [옵션] 실행:
- 매개변수 없음 - 전체 추적 보고서 표시
- `--brief` - 간략 정보 표시
- `--plot` - 플롯 추적만 표시
- `--stats` - 통계 데이터만 표시
- `--check` - **[강화]** 심층 일관성 검사 실행 (캐릭터 검증 포함)
- `--fix` - **[신규]** 발견된 간단한 문제 자동 수정

## 데이터 소스

여러 추적 파일의 정보를 통합:
- `progress.json` - 집필 진도
- `spec/tracking/plot-tracker.json` - 플롯 추적
- `spec/tracking/timeline.json` - 타임라인
- `spec/tracking/relationships.json` - 관계 네트워크
- `spec/tracking/character-state.json` - 캐릭터 상태
- `spec/tracking/validation-rules.json` - **[신규]** 검증 규칙 (--check 및 --fix용)

## 강화 기능

### 데이터 일관성 검증

기본 검사 (기본 실행):
- plot-tracker.json과 outline.md의 일관성
- timeline.json의 시간 논리
- relationships.json의 관계 충돌
- character-state.json의 위치 합리성

### 심층 검증 모드 (--check)

`--check` 매개변수 사용 시, 프로그래밍적 심층 검증 실행:

#### 내부 작업 흐름 (자동 실행)
```markdown
# Phase 1: 기본 검증 [병렬 실행]
- [x] T001 [P] 플롯 일관성 검사 실행 (plot-check 로직)
- [x] T002 [P] 타임라인 검증 실행 (timeline 로직)
- [x] T003 [P] 관계 검증 실행 (relations 로직)
- [x] T004 [P] 세계관 검증 실행 (world-check 로직)

# Phase 2: 캐릭터 심층 검증
- [x] T005 validation-rules.json 검증 규칙 로드
- [x] T006 모든 챕터의 캐릭터 이름 스캔
- [x] T007 character-state.json과 이름 일관성 비교
- [x] T008 호칭이 relationships.json에 부합하는지 검사
- [x] T009 캐릭터 행동이 인물 설정에 부합하는지 검증

# Phase 3: 종합 보고서 생성
- [x] T010 모든 검증 결과 종합
- [x] T011 문제 심각도 표기
- [x] T012 수정 제안 생성
```

#### 검증 보고서 예시
```
📊 심층 검증 보고서
━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 통과 항목: 15/18

❌ 발견된 문제 (3):
1. [높음] 제3장: 주인공 이름 "이명"은 "이중용"이어야 함
2. [중간] 제7장: 심옥경 호칭 오류, "사형"으로 사용
3. [낮음] 제12장: 타임라인 점프 미설명

🔧 자동 수정 가능: 2개
📝 수동 확인 필요: 1개

/track --fix로 간단한 문제를 자동 수정하세요
```

### 자동 수정 모드 (--fix)

`--fix` 매개변수 사용 시, 검증 보고서를 기반으로 자동 수정:

#### 자동 수정 범위
1. **캐릭터 이름 오류** - validation-rules.json에 따라 자동 교체
2. **고정 호칭 오류** - 올바른 호칭으로 자동 수정
3. **간단한 맞춤법 오류** - 명백한 오타 수정

#### 수정 흐름
```markdown
# 내부 수정 작업 (자동 실행)
- [x] F001 검증 보고서의 문제 목록 읽기
- [x] F002 [P] 제3장 캐릭터 이름 오류 수정
- [x] F003 [P] 제7장 호칭 오류 수정
- [x] F004 수정 보고서 생성
- [x] F005 추적 파일 업데이트
```

#### 수정 보고서 예시
```
🔧 자동 수정 보고서
━━━━━━━━━━━━━━━━━━━
✅ 수정 완료: 2개 문제
- 제3장: "이명" → "이중용"
- 제7장: "사형" → "공자"

⚠️ 수동 처리 필요: 1개 문제
- 제12장: 타임라인 점프에 대한 설명 보충 필요

수정 완료! /track --check를 다시 실행하여 검증하는 것을 권장합니다
```

### 스마트 분석 및 제안

1. **진도 분석**
   - 계획 진도와 실제 진도 비교
   - 완료 시간 예측
   - 집필 병목 구간 식별

2. **내용 분석**
   - 복선 커버리지 (배치됨/해소됨)
   - 캐릭터 등장 빈도
   - 갈등 강도 곡선

3. **행동 제안**
   분석 결과에 따라 제공:
   - 다음 집필 중점
   - 처리해야 할 복선
   - 강화 필요한 관계선
   - 타임라인 조정 제안
