# 신규 사용자 입문 가이드

## 활성화 조건

사용자가 다음과 같은 말을 할 때 자동 활성화:
- "소설을 쓰고 싶어요"
- "새 프로젝트를 시작하려고요"
- "novel-writer 어떻게 써요"
- "어디서부터 시작해요"

---

## 7단계 방법론 가이드

### 환영 메시지

```
👋 Novel Writer Skills에 오신 것을 환영합니다!

새 소설 프로젝트를 시작하시려는군요.
Novel Writer는 체계적인 7단계 방법론으로, 아이디어에서 완성까지 도와드립니다.

빠르게 프로세스를 안내해 드리겠습니다:

1단계: /constitution - 창작 헌법 생성
2단계: /specify - 스토리 규격 정의
3단계: /clarify - 모호한 부분 명확화 (5개 질문)
4단계: /plan - 창작 계획 수립
5단계: /tasks - 작업 체크리스트 분해
6단계: /write - 챕터 집필 실행
7단계: /analyze - 품질 검증 분석

각 단계를 자세히 알고 싶으신가요? 아니면 바로 1단계를 시작할까요?
```

### 단계별 상세 설명 (필요에 따라 제공)

```
【1단계: /constitution】
창작 헌법 생성 - 이것이 최고 원칙입니다.
- 핵심 가치관 (무엇을 전달하고 싶은가?)
- 품질 기준 (좋은 장이란 무엇인가?)
- 금기 레드라인 (쓸 수 없는 것은?)

왜 중요한가: 이후 모든 창작이 헌법에 부합해야 합니다.

【2단계: /specify】
스토리 규격 정의 - 무엇을 쓸지 명확화.
- 스토리 컨셉, 장르, 주제
- 타겟 독자, 분량, 결말
- 반드시 포함할 요소 (P0)

왜 중요한가: 규격은 기술 요구사항이며, 중간에 주제를 벗어나는 것을 방지.

【3단계: /clarify】
AI가 5개 핵심 질문을 제시하여, 모호한 부분을 명확화합니다.
- specification의 불명확한 부분을 대상으로
- 디테일을 생각하도록 강제

왜 중요한가: 모호한 규격 = 모호한 이야기.

【4단계: /plan】
창작 계획 수립 - 기술 방안.
- 장 구조, 리듬 설계
- 핵심 플롯 포인트 타임라인
- 기술적 난점의 해결 방안

왜 중요한가: 계획이 상세해야, 실행이 순조롭습니다.

【5단계: /tasks】
작업 체크리스트 분해 - 실행 가능한 단계.
- 각 장마다 하나의 작업
- 상태 표시 (pending/in_progress/completed)
- 진도 추적

왜 중요한가: 큰 목표를 작은 작업으로 분해하여 부담 줄이기.

【6단계: /write】
집필 실행 - 작업 체크리스트에 기반.
- 모든 context를 자동 로드
- 헌법, 규격, 계획을 준수
- 품질 자체점검 후 저장

왜 중요한가: 7단계 방법론의 장점이 집중적으로 드러나는 곳.

【7단계: /analyze】
품질 검증 - 일관성 확보.
- 프레임워크 검사 (구조, 리듬)
- 내용 검사 (일관성, 품질)
- 문제 발견, 수정 제안

왜 중요한가: 정기적 분석으로, 적시에 조정하고, 대규모 재작업을 피함.
```

## 자주 묻는 질문 빠른 답변

### Q: 반드시 순서대로 해야 하나요?

**A**: 순서대로 권장하지만, 강제는 아닙니다.

```
권장 프로세스: 1→2→3→4→5→6→7 (6-7 반복)

건너뛰기 허용:
- 경험 많은 작가는 /clarify 건너뛰기 가능
- 단편은 /plan 간소화 가능

비권장:
- 바로 /write로 건너뛰기 (규격과 계획 없으면, 쓰다가 혼란)
```

### Q: 7단계 완료까지 얼마나 걸리나요?

**A**: 이야기 복잡도에 따라 다릅니다.

```
간단한 이야기 (5만자 단편):
- 1-5단계: 2-3시간
- 6-7단계: 집필 실행, 계속 진행

복잡한 이야기 (50만자 장편):
- 1-5단계: 1-2일
- 6-7단계: 수개월

초기 투자가 후기의 순조로움을 가져옵니다.
```

### Q: 중간에 수정할 수 있나요?

**A**: 물론이죠!

```
규격이 맞지 않다면:
→ specification.md 수정
→ /plan을 다시 실행하여 계획 조정

계획을 조정해야 한다면:
→ creative-plan.md 수정
→ tasks.md 업데이트

이것은 반복 과정이지, 한 번에 확정하는 것이 아닙니다.
```

## 첫 사용 시 조언

### 간단한 프로젝트부터 시작

```
✓ 권장:
- 5-10만자의 중단편
- 단일 장르 (로맨스/미스터리/역사)
- 간단한 플롯 (1-2개 메인 라인)

⚠️ 첫 번째에 비권장:
- 50만자+ 장편
- 다중 장르 융합 (로맨스+미스터리+판타지+역사)
- 초복잡 세계관

먼저 프로세스에 익숙해지고, 복잡한 프로젝트에 도전하세요.
```

### 첫 번째 명령 실행

```
준비되셨나요? 1단계를 시작합시다:

명령 실행: /constitution

창작 헌법 생성을 안내해 드리겠습니다.
다음 질문에 답할 준비를 하세요:
1. 이 이야기로 어떤 핵심 이념을 전달하고 싶은가요?
2. 품질 기준은 무엇인가요?
3. 절대 쓸 수 없는 내용은 무엇인가요?

준비되면 입력하세요: /constitution
```
