# 글쓰기 스타일 탐지기

## 핵심 기능

사용자의 글쓰기 스타일 요구를 자동 탐지하고, 해당하는 스타일 지식 베이스를 원활하게 로드합니다.

## 탐지 가능한 스타일

### 1. natural-voice (자연스러운 목소리 스타일)
- **적합**: 도시 현대소설, 현실 소재, 감정류 소설
- **특징**: 구어체, 생활감, 진실감 강함
- **트리거 키워드**: 구어체, 진실감, 현실감 있는, 말하듯이

### 2. literary (문학 스타일)
- **적합**: 순수문학 작품, 역사소설, 사상적 깊이가 필요한 이야기
- **특징**: 아름다움, 깊이, 문학성, 의경 중시
- **트리거 키워드**: 문학성, 순수문학, 깊이 있는

### 3. web-novel (웹소설 스타일)
- **적합**: 판타지소설, 도시 쾌감물, 시스템물, 회귀물
- **특징**: 빠른 리듬, 갈등 밀집, 쾌감 빈번
- **트리거 키워드**: 웹소설, 쾌감물, 사이다, 역전

### 4. ancient-style (고풍)
- **적합**: 무협소설, 선협, 고대 배경 소설
- **특징**: 고운이 있되 순수 고전문이 아님, 이미지 풍부, 시적
- **트리거 키워드**: 고풍, 고운, 무협풍, 강호 느낌

### 5. minimal (미니멀 스타일)
- **적합**: 추리 미스터리, 하드보일드 소설, 실험적 작품
- **특징**: 고도로 간결, 헤밍웨이식 절제, 여백의 미학
- **트리거 키워드**: 미니멀, 헤밍웨이, 절제, 하드보일드

상세 키워드 목록과 트리거 시나리오는 [KEYWORDS.md](KEYWORDS.md)를 참조하세요.

## 활성화 후 워크플로

### 단계 1: 스타일 선택 확인

키워드 탐지 후, 먼저 사용자 의도를 확인합니다:

```
【구어체】를 언급하신 것을 감지했습니다. 이것은 보통 natural-voice 스타일에 해당합니다.

다음을 할 수 있습니다:
1. 📖 이 스타일의 상세 가이드 보기
2. ✅ 이 스타일을 프로젝트에 직접 적용
3. 🔍 다른 스타일 옵션 확인

어떤 것을 원하시나요?
```

### 단계 2: 스타일 지식 베이스 로드

사용자가 확인하면, Read 도구로 해당 지식 베이스 파일을 읽습니다:

```
📖 읽기: .claude/knowledge-base/styles/{style-name}.md
```

핵심 요점을 표시하고, 전체 규범은 이미 컨텍스트에 로드됩니다.

### 단계 3: 설정 마커 작성

Edit 도구로 사용자의 `specification.md`에 설정을 추가합니다:

```yaml
---
writing-style: natural-voice
---
```

사용자에게 설정이 저장되었으며, /write 실행 시 자동으로 해당 스타일 규범을 따른다고 안내합니다.

## 다중 스타일 처리

여러 스타일 키워드가 탐지되면, 호환성 검사를 수행합니다:

- **호환 가능한 조합**: 혼합 방안과 비율 제안 제공
- **충돌하는 조합**: 모순을 알리고 해결 방안 제시

상세한 충돌 처리 로직은 [CONFLICT_RESOLUTION.md](CONFLICT_RESOLUTION.md)를 참조하세요.

## 능동적 제안

- **스토리 유형 기반 스타일 제안**: specification의 스토리 유형을 분석하여, 적합한 스타일을 능동적으로 추천
- **중도 조정 지원**: 사용자 피드백을 지속적으로 모니터링하며, 스타일 조정을 지원

## requirement-detector와의 협력

본 Skill은 문체 레이어에 집중하며, requirement-detector(규범 레이어)와 협동합니다:

```
style-detector → 문체 식별 → styles/*.md 로드
requirement-detector → 규범 식별 → requirements/*.md 로드

최종 조합 효과 예시:
natural-voice (스타일) + anti-ai-v4 + fast-paced (규범)
= 자연스러운 목소리의 빠른 속도 쾌감물, 강력한 AI 느낌 제거
```
