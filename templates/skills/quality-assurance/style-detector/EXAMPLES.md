# 완전한 대화 예시와 성공 사례

본 문서는 style-detector의 실제 사용 시 완전한 대화 흐름을 보여줍니다.

---

## 예시 1: 기본 스타일 탐지

**사용자**:
> "도시 쾌감물을 쓰려는데, 구어체로, 진짜 같이"

**style-detector 응답**:
```
🎨 스타일 탐지기 활성화

감지된 키워드:
- "구어체" → natural-voice
- "진짜 같이" → natural-voice

제안 스타일: **natural-voice** (자연스러운 목소리)

이 스타일의 특징:
- 구어체 표현 위주, 친구에게 이야기하듯이
- 생활 디테일이 풍부, 몰입감 강함
- 대화로 플롯 추진 (비율 40-60%)
- 문학적 어투와 서면어 회피

상세 스타일 가이드를 보시겠습니까?
```

**사용자**:
> "좋아요, 이 스타일을 적용해주세요"

**style-detector 실행**:
```
✅ 스타일 지식 베이스 로딩 중...

📖 읽기: .claude/knowledge-base/styles/natural-voice.md

【핵심 요점 표시】

## 핵심 원칙:
1. 구어체 우선 - 5글자로 설명할 수 있으면, 10글자를 쓰지 않음
2. 대화로 플롯 추진 - 대화 비율 40-60%
3. 행동>심리 - 무엇을 했는지 많이 쓰고, 무엇을 생각했는지 적게
4. 구체적>추상적 - 구체적 디테일, 추상적 개념은 안 됨

## AI 느낌 제거 요점:
- 금지어: 그러나, 그야말로, 만감이 교차하다, 극히...
- 형용사 제한: 100자당 최대 5개
- 사자성어 사용: 안 쓸 수 있으면 안 씀

✅ 스타일 설정을 specification.md에 작성 완료

/write 실행 시, natural-voice의 모든 규범을 자동으로 엄격히 준수합니다.

💡 배합 제안:
"쾌감물"도 언급하셨으니, requirement-detector를 활성화하여
fast-paced 규범을 로드하는 것도 추천합니다.
```

## 예시 2: 스타일 능동 제안

**사용자가 specification.md에 작성**:
> "유형: 도시 리얼리즘 소설, 보통 사람의 삶을 다룸"

**style-detector 능동적 제안**:
```
💡 스타일 제안

스토리 유형(도시 리얼리즘, 보통 사람의 삶)에 따라,
제안 사용: **natural-voice** (자연스러운 목소리 스타일)

이유:
✅ 현실 소재는 구어체에 적합
✅ 보통 사람의 삶에는 진실감이 필요
✅ 몰입감 강함

물론, 이것은 제안일 뿐이고, 다른 스타일도 선택할 수 있습니다.
natural-voice 스타일을 적용하시겠습니까?
```

## 예시 3: 스타일 혼합

**사용자**:
> "역사소설을 쓰려는데, 고풍 느낌이 있되, 문학성도 유지하고 싶어요"

**style-detector 응답**:
```
🔍 다중 스타일 탐지 결과

감지됨:
- ancient-style ("고풍 느낌" 때문에)
- literary ("문학성" 때문에)

이 두 스타일은 아주 잘 조합됩니다!

제안 방안:
1. 메인 스타일: ancient-style (60%) + 보조: literary (40%)
   - 고운으로 분위기 조성
   - 문학성으로 의경 심화

2. 메인 스타일: literary (55%) + 보조: ancient-style (45%)
   - 문학성을 기반으로
   - 고풍 요소를 곁들임

추천: 방안1이 역사소설에 더 적합

이 조합을 적용하시겠습니까?
```

## 예시 4: 중도 조정

**프로젝트가 10장까지 진행된 후 사용자 발언**:
> "최근 몇 장이 너무 문학적이에요, 독자가 이해하기 어렵다는 피드백이 있어서, 좀 더 쉽게 바꿀 수 있나요"

**style-detector 응답**:
```
🎨 스타일 조정 탐지

스타일 조정 요구를 감지했습니다:
현재 스타일: literary
제안 새 스타일: natural-voice

이것은 후속 장의 집필 스타일에 영향을 미칩니다.

조정 제안:
1. 11장부터 natural-voice 사용
2. 1-10장은 수정하지 않음 (이전 글 일관성 유지)

이렇게 하면 좋은 점:
✅ 후속 장이 더 읽기 쉬움
✅ 이미 작성된 내용의 완전성 유지

이렇게 하면 위험한 점:
⚠️ 전후 스타일에 명확한 변화가 있음
⚠️ 독자가 문체 전환을 느낄 수 있음

조정을 확인하시겠습니까? 아니면 더 완만한 전환 방안을 원하시나요?
```

## 예시 5: requirement-detector와 협동

**사용자**:
> "현대 로맨스를 쓰려는데, 달달하게, 구어체로, AI 느낌도 빼주세요"

**system (두 detector 협동)**:
```
🎨 style-detector 활성화
감지됨: 구어체 → natural-voice

📋 requirement-detector 활성화
감지됨: 달달하게 → romance-sweet
감지됨: AI 느낌 빼기 → anti-ai-v4

✅ 추천 설정 조합:

스타일 레이어: natural-voice
규범 레이어: romance-sweet + anti-ai-v4

최종 효과:
자연스러운 구어체의 달콤한 현대 로맨스, 강력한 AI 느낌 제거

설정 작성 완료:
---
writing-style: natural-voice
writing-requirements:
  - romance-sweet
  - anti-ai-v4
---

이 조합이 요구에 특히 적합합니다. 적용을 확인하시겠습니까?
```

---

## 사용 팁

1. **키워드를 명확히**: "구어체", "웹소설", "고풍" 등의 트리거 키워드를 직접 말하세요
2. **구체적 요구 설명**: 스토리 유형, 타겟 독자를 설명
3. **제안을 신뢰**: 탐지기가 경험에 기반하여 추천합니다
4. **과감하게 조정**: 집필 과정에서 언제든 스타일 조정 가능
5. **조합 사용**: 스타일(style) + 규범(requirement) 조합이 더 효과적

## 자주 묻는 질문

**Q: 여러 스타일을 동시에 사용할 수 있나요?**
A: 가능하지만, 호환성에 주의하세요. [CONFLICT_RESOLUTION.md](CONFLICT_RESOLUTION.md)에서 호환 매트릭스를 참조하세요.

**Q: 중간에 스타일을 바꿀 수 있나요?**
A: 가능하지만, 새 장부터 시작하는 것을 권장하며, 이미 작성된 내용의 일관성을 유지하세요.

**Q: 어떤 스타일을 써야 할지 모르겠어요?**
A: 스토리 유형과 타겟 독자를 설명하시면, 탐지기가 능동적으로 제안합니다.

**Q: 스타일과 규범은 무슨 차이인가요?**
A: 스타일(style)은 문체와 어감을 조절하고, 규범(requirement)은 집필 기준과 요구사항을 조절합니다. 둘 다 조합하여 사용할 수 있습니다.
