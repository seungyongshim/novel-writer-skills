# 다중 스타일 탐지와 충돌 처리

여러 스타일 키워드가 탐지되면, 충돌 검사를 수행하고 해결 방안을 제공해야 합니다.

---

## 시나리오 1: 호환 가능한 조합

### 예시: natural-voice + literary

**사용자 발언**:
> "도시소설을 쓰려는데, 구어체로 하되, 문학성도 좀 있으면 좋겠어요"

**탐지 결과**:
- natural-voice (구어체)
- literary (문학성)

**처리 방식**:
```markdown
🔍 다중 스타일 탐지 결과

감지됨:
- natural-voice ("구어체" 때문에)
- literary ("문학성" 때문에)

이 두 스타일은 혼합 사용이 가능합니다!

제안 방안:
1. 메인 스타일: natural-voice (70%) + 보조: literary (30%)
   - 대화와 일상은 구어체
   - 핵심 순간에 문학적 묘사 추가

2. 메인 스타일: literary (60%) + 보조: natural-voice (40%)
   - 전체적으로 문학성 유지
   - 대화 부분은 구어체화

어떤 비율을 선호하시나요?
```

## 시나리오 2: 충돌하는 조합

### 예시: minimal + literary

**사용자 발언**:
> "미니멀 스타일인데, 화려한 수사도 넣고 싶어요"

**탐지 결과**:
- minimal (미니멀)
- literary (화려함) - 충돌!

**처리 방식**:
```markdown
⚠️ 스타일 충돌 감지

감지됨:
- minimal (미니멀 스타일) - 모든 수식 제거 요구
- literary (문학 스타일) - 아름다운 언어 필요

이 두 스타일은 충돌합니다:
- 미니멀 요구: 형용사 제거, 극도의 짧은 문장, 수식 제로
- 문학 요구: 정확한 어휘, 미감, 적절한 수식

제안:
1. minimal 선택, 화려한 수사 포기
2. literary 선택, "미니멀"을 "간결"로 조정
3. natural-voice 고려 (두 가지의 중간)

어떤 방향을 선호하시나요?
```

## 충돌 매트릭스

| 조합 | 호환성 | 처리 방식 |
|------|--------|----------|
| natural-voice + literary | ✅ 호환 | 비율 혼합 방안 제공 |
| natural-voice + web-novel | ✅ 호환 | 구어체 웹소설, 추천 |
| natural-voice + ancient-style | ⚠️ 주의 | 현대 구어 vs 고운, 균형 필요 |
| natural-voice + minimal | ⚠️ 주의 | 둘 다 간결 강조, 조합 가능 |
| literary + web-novel | ⚠️ 주의 | 문학성 vs 쾌감물 패턴, 취사선택 필요 |
| literary + ancient-style | ✅ 호환 | 둘 다 의경 중시, 추천 |
| literary + minimal | ❌ 충돌 | 화려함 vs 미니멀, 모순 |
| web-novel + ancient-style | ⚠️ 주의 | 빠른 리듬 vs 고운 여백, 균형 필요 |
| web-novel + minimal | ❌ 충돌 | 패턴화 vs 미니멀리즘, 모순 |
| ancient-style + minimal | ⚠️ 주의 | 이미지 vs 절제, 시도 가능 |

## 중도 스타일 조정

프로젝트가 이미 시작된 후에도, 사용자 피드백을 지속적으로 모니터링합니다.

### 시나리오: 중도 스타일 전환

**사용자가 10장에서 발언**:
> "최근 몇 장이 너무 문학적이에요, 구어체로 바꾸고 싶어요"

**탐지기 활성화**:
```markdown
🎨 스타일 조정 탐지

스타일 조정 요구를 감지했습니다:
현재 스타일: literary
제안 새 스타일: natural-voice

이것은 후속 장의 집필에 영향을 미칩니다.

조정 제안:
1. 다음 장부터 새 스타일 사용
2. 이미 작성된 장은 수정하지 않음 (일관성 유지)

조정을 확인하시겠습니까?
```

## 처리 원칙

1. **사용자 의도 우선**: 사용자가 명확히 선호를 표현하면, 사용자 선택을 존중
2. **옵션 제공**: 충돌 발생 시, 2-3개 해결 방안 제시
3. **이유 설명**: 왜 충돌하는지 설명하여, 사용자의 이해를 도움
4. **혼합 지원**: 호환 가능한 조합에 대해, 비율 제안 제공
5. **조정 허용**: 중도 전환을 지원하되, 영향 범위를 안내
