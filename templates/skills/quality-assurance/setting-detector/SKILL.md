# 스토리 설정 자동 감지기

## 핵심 기능

**지식 베이스 자동 활성화 시스템** - Novel Writer Skills의 핵심 경쟁력입니다.

특정 키워드를 언급하면, 자동으로:
1. 이야기의 장르, 시대, 주제를 감지
2. 해당하는 집필 지식 베이스를 로드
3. 전체 창작 과정에서 전문 지식을 적용

**수동 호출 불필요** - 완전 자동화, 백그라운드 실행.

## 자동 활성화 흐름

### 예시 1: 단일 장르 감지

```
사용자: "로맨스 소설을 쓰고 싶어요"
              ↓
[키워드 감지: "로맨스"]
              ↓
✓ 자동 로드: romance.md
              ↓
AI 응답: "좋습니다! 로맨스 소설 창작을 도와드리겠습니다.
로맨스 장르 관습에 따라, 몇 가지 핵심 요소를 명확히 해야...
(romance.md의 지식을 자동 적용)"
```

### 예시 2: 다중 장르 조합 감지

```
사용자: "1920년대 배경의 로맨스 복수 소설을 쓰려고요"
              ↓
[키워드 감지: "1920", "로맨스", "복수"]
              ↓
✓ 자동 로드: romance.md
✓ 자동 로드: revenge.md
✓ 자동 로드: references/china-1920s/
              ↓
📚 활성화된 지식 베이스:
- genres/romance.md (로맨스 소설 관습)
- genres/revenge.md (복수물 기법)
- references/china-1920s/ (1920년대 배경)
              ↓
AI 응답: "훌륭한 조합입니다! 로맨틱 서스펜스 + 복수 + 민국 배경이군요.
세 장르의 융합에 따라 제안드리면...
(세 지식 베이스의 내용을 동시 적용)"
```

### 예시 3: 창작 과정의 지속적 적용

```
/constitution 단계:
→ 활성화: romance.md
→ 알림: 로맨스 소설은 HEA/HFN 결말 약속이 필요

/specify 단계:
→ 활성화: romance.md + revenge.md
→ 제안: 관계 아크 + 복수 목표 정의

/plan 단계:
→ 활성화: 모든 지식 베이스
→ 적용: 감정 비트 포인트 + 사이다 리듬 + 1920년대 디테일

/write 단계:
→ 활성화: 모든 지식 베이스
→ 실시간: 대화 기법 + 장면 묘사 + 시대 분위기

/analyze 단계:
→ 활성화: 모든 지식 베이스
→ 검사: 로맨스 관습 + 복수 합리성 + 역사적 정확성
```

## Token 효율 최적화

**왜 이 시스템이 효율적인가?**

전통 방식 (superpowers-skills 모드):
```
50개 독립 Skills × 각 40행 SKILL.md = 2000 tokens
매 대화마다 모든 Skills 로드
```

우리의 방식 (계층적 지식 베이스 + 감지기):
```
1개 detector Skill (본 파일): ~100 tokens
필요에 따라 지식 베이스 로드:
- romance.md: ~520 tokens
- revenge.md: ~480 tokens
- china-1920s/: ~650 tokens

총계: 100 + 520 + 480 + 650 = ~1750 tokens
(관련 지식 베이스만 로드)
```

무관한 50개 Skills 로드 대비 **75% 절약**

## 수동 제어

### 현재 활성화된 지식 베이스 확인

언제든 질문:
```
"현재 어떤 지식 베이스가 활성화되어 있나요?"
```

응답 예시:
```
📚 현재 활성화된 지식 베이스:
✓ genres/romance.md - 로맨스 소설 창작 관습
✓ genres/revenge.md - 복수물 사이다 기법
✓ references/china-1920s/ - 1920년대 중국 참고 자료

Token 소비: 약 1650 tokens
```

### 수동으로 지식 베이스 활성화

자동 감지가 실패하면, 명시적으로 지정 가능:
```
"romance과 mystery 지식 베이스를 로드해 주세요"
"이 이야기에 1920년대 중국 참고 자료가 필요해요"
"무협 지식 베이스를 활성화해 주세요"
```

### 지식 베이스 비활성화

특정 지식 베이스가 더 이상 필요 없으면:
```
"revenge 지식 베이스를 비활성화해 주세요, 이 이야기는 복수와 관련 없어요"
```

## Commands와의 통합

### `/constitution` - 장르별 원칙

활성화된 지식 베이스에 따라, 장르별 창작 원칙 제공:

```markdown
【constitution에 자동 삽입】

## 장르별 원칙 (setting-detector가 자동 추가)

### 로맨스 소설 핵심 원칙
1. 관계 아크가 메인 플롯
2. HEA/HFN 결말은 필수
3. 케미가 진실되고 믿을 수 있게

### 복수물 핵심 원칙
1. 원한이 깊고 구체적이어야
2. 사이다 리듬은 점진적으로
3. 도덕적 균형: 죄값을 치름
```

### `/specify` - 장르별 규격

해당하는 규격 템플릿 요소 제공:

```markdown
【자동 안내】

💡 활성화된 지식 베이스에 따라, 규격에 정의를 제안합니다:

【romance.md에서】
- 관계 아크 설정
- 감정 비트 포인트 계획
- HEA/HFN 결말 유형

【revenge.md에서】
- 원한의 원천 (구체적 죄행)
- 치트키 설정
- 사이다 리듬 계획

【china-1920s에서】
- 구체적 연도 (1920-1929)
- 지리적 위치 (베이징/상하이/기타)
- 사회 계층 설정
```

### `/plan` - 장르 지식 융합

여러 지식 베이스의 제안을 계획에 융합:

```markdown
【자동 적용】

## 장 계획 (3개 지식 베이스 융합)

1-5장: 만남과 원한 확립
- [romance] 첫 만남 장면 (meet-cute)
- [revenge] 전생/과거의 깊은 원한 전시
- [1920s] 민국 시대 분위기 조성

6-10장: 관계 진전과 첫 사이다
- [romance] 썸 기간: 스킨십, 감정 축적
- [revenge] 첫 사이다: 원수의 놀라움
- [1920s] 복식, 예법, 사회 규칙 전시

...
```

### `/write` - 실시간 집필 가이드

집필 시 관련 지식 자동 적용:

```markdown
【집필 전 자동 알림】

📋 본 장에 적용되는 지식 베이스:

✓ romance.md - 대화 스파크 기법
  → 밀당 대화: 순종이 아닌 도전
  → 바디랭귀지: 시선, 접촉, 거리

✓ revenge.md - 사이다 장면 쾌감
  → 기대 → 반전 → 경악 → 통쾌

✓ 1920s - 시대 디테일
  → 호칭: 나리, 도련님, 아가씨
  → 장소: 찻집, 양행, 조계
```

### `/analyze` - 장르 관습 검사

활성화된 지식 베이스에 따라 해당 검사 실행:

```markdown
【자동 분석】

## 장르 부합도 분석

### 로맨스 소설 검사 (romance.md 기반)
- [x] 관계 아크가 메인 플롯
- [x] 필수 감정 비트 포인트 포함
- [ ] ⚠️ 첫 키스 장면 누락 (15장에 제안)
- [x] HEA 결말 약속 명확

### 복수물 검사 (revenge.md 기반)
- [x] 원한이 깊고 구체적
- [x] 사이다 빈도 합리적 (매 5장 1회)
- [x] 주인공 실력 향상 합리적

### 역사 정확성 검사 (1920s 기반)
- [x] 시대 배경 올바름
- [ ] ⚠️ 8장에 "스마트폰" 등장 (시대에 맞지 않음)
- [x] 호칭 체계 올바름
```

### 현재 지원하는 지식 베이스

| 카테고리 | 완료 | 계획 중 |
|---------|------|--------|
| 장르 지식 | 5개 | 10+ |
| 참고 자료 | 1개 | 20+ |

**완료** (v1.0):
- genres/romance.md
- genres/mystery.md
- genres/historical.md
- genres/revenge.md
- genres/wuxia.md
- references/china-1920s/

**계획 중** (v1.1+):
- genres/fantasy.md (판타지)
- genres/sci-fi.md (SF)
- genres/horror.md (공포)
- references/ancient-china/ (각 왕조)
- references/modern-workplace/ (현대 직장)

### 새 지식 베이스 추가

1. `templates/knowledge-base/` 해당 디렉토리에 파일 생성
2. `templates/knowledge-base/README.md`의 키워드 매핑 테이블 업데이트
3. setting-detector가 자동 인식 (본 파일 수정 불필요)

**예시**: 판타지 지식 베이스 추가

```bash
# 1. 파일 생성
touch templates/knowledge-base/genres/fantasy.md

# 2. README.md 키워드 매핑 업데이트
fantasy:
  keywords: [판타지, 마법, 세계관 구축, 마법 시스템]
  auto_load: genres/fantasy.md

# 3. 완료! 다음에 사용자가 "판타지"라고 하면 자동 활성화
```

## 지능형 특성

### 1. 퍼지 매칭

정확한 키워드가 아니어도 인식:

```
"민국 시대로 회귀해서 복수하는 이야기를 쓰고 싶어요"
           ↓
인식: "민국" → 1920년대 중국
     "복수" → revenge

"주인공이 회귀 후 보복해요"
           ↓
인식: "회귀" + "보복" → revenge (회귀 복수)
```

### 2. 컨텍스트 이해

대화 컨텍스트에 따라 지속 인식:

```
첫 번째 메시지: "소설을 쓰고 싶어요"
→ 어떤 지식 베이스도 활성화하지 않음 (더 많은 정보 대기)

두 번째 메시지: "주인공은 탐정이에요"
→ mystery.md 활성화

세 번째 메시지: "감정선도 있어요"
→ romance.md 추가 활성화

→ 현재 활성화: mystery + romance (로맨틱 미스터리)
```

### 3. 자동 중복 제거

중복 활성화 방지:

```
사용자: "이건 무협소설이에요, 강호 배경, 무공이 나와요"
            ↓
무협 키워드 3개 감지하지만, wuxia.md는 1회만 활성화
```

## 자주 묻는 질문

### Q: 어떤 지식 베이스가 활성화되었는지 어떻게 아나요?

**A**: 언제든 "현재 활성화된 지식 베이스가 뭐예요?"라고 물어보세요. 목록을 알려드립니다.

### Q: 자동 감지가 틀리면?

**A**: 직접 알려주세요:
```
"이건 로맨스 소설이 아니에요, romance 지식 베이스를 비활성화해 주세요"
"이건 SF예요, sci-fi 지식 베이스를 활성화해 주세요" (있는 경우)
```

### Q: 동시에 몇 개까지 활성화할 수 있나요?

**A**: 이론상 무제한이지만, 3-5개가 최적:
- 너무 적으면: 가이드가 불충분
- 너무 많으면: token 소비 증가, 충돌 가능

실제 경험: 3개 지식 베이스 (예: romance + revenge + 1920s)로 이미 풍부.

### Q: 지식 베이스끼리 충돌하나요?

**A**: 아닙니다! 지식 베이스는 협업하도록 설계되었습니다:
- romance + mystery = 로맨틱 미스터리 ✓
- fantasy + romance = 판타지 로맨스 ✓
- historical + revenge = 사극 복수물 ✓

### Q: 수동으로 지정해야 하는 경우는?

**A**: 다음 경우:
- 매우 전문적인 용어를 사용 (감지 실패 가능)
- 다른 장르의 융합을 시도하고 싶을 때
- 자동 감지가 의도와 맞지 않을 때

### Q: 지식 베이스가 계속 활성화되나요?

**A**: 네, 다음까지:
- 명시적으로 "XX 지식 베이스 비활성화"라고 할 때
- 또는 새 대화를 시작할 때
- 또는 완전히 다른 이야기로 전환할 때

## 모범 사례

### 1. 창작 초기에 장르를 명확히

```
권장:
사용자: "민국 배경의 복수 로맨스 소설을 쓸 거예요"

비권장:
사용자: "소설 쓸래요"
AI: "네"
사용자: "주인공 성은 이씨예요"
AI: "네"
(20턴 후에야 어떤 장르인지 언급)
```

### 2. 정기적으로 활성화 상태 확인

매 5-10장마다:
```
"현재 활성화된 지식 베이스가 아직 적합한가요?"
```

검사 후 조정 필요 여부를 제안합니다.

### 3. 장르 융합은 합리적으로

```
✓ 좋은 조합:
- romance + mystery (로맨틱 미스터리)
- historical + romance (사극 로맨스)
- revenge + romance (복수 + 감정선)

⚠️ 어려운 조합:
- horror + romance (공포 로맨스? 독자가 불편할 수 있음)
- mystery + wuxia (추리 무협, 특별한 설계 필요)

불가능하진 않지만, 융합 방식에 특히 주의가 필요합니다.
```

### 4. 참고 자료 활용

이야기에 명확한 시대 배경이 있으면, 반드시 해당 참고 자료 활성화:

```
"1920년대" → china-1920s
"고대" → 구체적 왕조 지정 (예: "당나라", "조선")
```

역사 디테일이 정확하면, 독자 몰입감이 강해집니다.

## 기술 설명

### 감지 알고리즘

```
1. 사용자 입력 → 키워드 추출
2. 키워드 → 지식 베이스에 매핑 (README.md 기반)
3. 중복 제거 → 중복 활성화 방지
4. 로드 → 해당 .md 파일 Read
5. 적용 → 후속 대화에서 지속적으로 지식 적용
```

### Token 관리

```
활성화 상태 검사: 매 대화 시작 시
지식 베이스 내용: 필요에 따라 로드, 대화 컨텍스트에 캐시
업데이트 전략: 명확히 필요할 때만 재읽기
```

### 다른 Skills과의 협업

setting-detector는 "지식 디스패처", 다른 Skills은 "실행자":

```
setting-detector → 어떤 지식을 로드할지 결정
consistency-checker → 활성화된 지식 베이스 기반 일관성 검사
workflow-guide → 장르에 따른 특정 프로세스 안내
genre-specific skills → 장르별 심층 기법 제공
```
