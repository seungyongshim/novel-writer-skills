# 잊힌 요소 알림기

## 핵심 기능

**장편소설의 흔한 문제를 방지**:
- 캐릭터가 갑자기 사라짐 ("조연 A는 어디 갔어?")
- 플롯 라인이 잊힘 ("그 복선 나중에 어떻게 됐어?")
- 복선이 회수되지 않음 ("앞에서 말한 보물은?")

**해결 방안**: 백그라운드에서 모니터링하고, 오랫동안 등장하지 않은 요소를 능동적으로 알림.

---

## 모니터링 내용

### 1. 캐릭터 등장 빈도

```
모니터링: character-state.json의 모든 캐릭터
임계값: 10장 미등장

알림 예시:
⚠️ 캐릭터 알림:
"조연 이명"이 10장 동안 미등장 (마지막: 5장)
- 다시 등장시켜야 할까요?
- 아니면 이 캐릭터의 이야기는 끝났나요?
```

### 2. 플롯 라인 진행도

```
모니터링: plot-tracker.json의 활성 플롯 라인
임계값: 12장 무진전

알림 예시:
⚠️ 플롯 라인 알림:
"아버지 원수 찾기" 플롯 라인이 12장 무진전
- 마지막 추진: 8장
- 현재 상태: pending
- 제안: 관련 플롯을 배치하거나 일시정지로 표시
```

### 3. 복선 회수

```
모니터링: 앞 글에서 깔아놓은 중요 복선
임계값: 20장 미회수

알림 예시:
⚠️ 복선 알림:
3장에서 언급한 "신비한 상자"가 아직 미공개 (23장 경과)
- 독자가 이미 잊었을 수 있음
- 제안: 빠른 시일 내 회수하거나 대화에서 언급
```

---

## 알림 타이밍

### 집필 전 알림

```
`/write` 실행 시, 잊힌 요소가 감지되면:

📋 집필 전 검사...
⚠️ 잊힌 요소 3개 발견:

1. 캐릭터 "장 아줌마"가 15장 미등장
2. 플롯 라인 "보물찾기"가 13장 무진전
3. 복선 "신비한 편지"가 20장 미공개

💡 이번 장에서 처리를 고려하시겠습니까?
```

### 분석 시 종합

```
`/analyze` 실행 시, 완전한 보고서 생성:

## 잊힌 요소 보고서

### 실종 캐릭터 (3명)
1. 장 아줌마 (15장 미등장)
2. 이 사장 (11장 미등장)
3. 왕 의사 (10장 미등장)

### 정체된 플롯 라인 (2개)
1. 보물찾기 라인 (13장 무진전)
2. 복수 라인 (12장 무진전)

### 미회수 복선 (1개)
1. 신비한 편지 (20장 미공개)

제안:
- 실종 캐릭터 우선 처리 (독자가 이미 잊었을 수 있음)
- 정체된 플롯 라인을 추진하거나 "일시정지"로 표시
- 복선을 적시에 회수 (그렇지 않으면 미완 결말)
```

## 설정 옵션

### 임계값 조정

```
"캐릭터 미등장 알림 임계값을 15장으로 변경"
"플롯 라인 정체 알림 임계값을 20장으로 변경"
```

### 특정 요소 제외

```
"장 아줌마 캐릭터는 완결됨, 더 이상 알리지 마세요"
"보물찾기 라인은 의도적으로 일시정지, 30장 후에 추진, 알리지 마세요"
```

## 모범 사례

1. **정기 확인**: 매 10장마다 `/analyze`를 실행하여 보고서 확인
2. **적시 처리**: 너무 많은 요소가 쌓이지 않도록
3. **능동적 표시**: 완결된 캐릭터는 "retired"로 표시
