# 글쓰기 규범 탐지기

## 핵심 기능

사용자의 글쓰기 규범 요구를 자동 탐지하고, 해당하는 규범 지식 베이스를 원활하게 로드합니다.

## 탐지 가능한 규범

### 1. anti-ai-v4 (강력 AI 느낌 제거)
- **적합**: AI 느낌에 제로 톨러런스인 작품, 극도의 구어체가 필요한 소설
- **특징**: 200+ 금지 어휘 목록, 6단계 규칙 체계, 극도의 자연화
- **트리거 키워드**: AI 느낌 강함, AI 느낌 제거, 너무 기계적, 부자연스러움, 최강 제거

### 2. anti-ai-v3 (표준 AI 느낌 제거)
- **적합**: 대부분의 현대소설, 자연감과 문학성의 균형이 필요한 경우
- **특징**: 균형 잡힌 AI 제거 전략, 적절한 어휘 제한
- **트리거 키워드**: 자연스럽게, AI 느낌 줄여서, 적절히 제거, 표준 제거

### 3. fast-paced (빠른 리듬/쾌감물 규범)
- **적합**: 도시 쾌감물, 판타지 레벨업물, 시스템물, 회귀물
- **특징**: 매 장 최소 2개 쾌감 포인트, 역전/사이다/레벨업 공식
- **트리거 키워드**: 쾌감물, 빠른 리듬, 사이다, 역전, 레벨업

### 4. no-poison (무독소 규범)
- **적합**: 모든 유형 소설 (범용), 논리성을 중시하는 작품
- **특징**: 지력저하, 강제 오해, 먼치킨, 도구인, 논리 허점 방지
- **트리거 키워드**: 무독소, 지력저하 없이, 논리 자족, 합리적, 강제 없이

### 5. serious-literature (순수문학 규범)
- **적합**: 리얼리즘 소설, 역사소설, 사회 문제 소설
- **특징**: 리얼리즘 수법, 사상성과 철학적 탐구, 언어 절제
- **트리거 키워드**: 순수문학, 깊이 있는, 사상성, 인간 본성 탐구

### 6. strong-emotion (강한 감정 규범)
- **적합**: 감정류 소설, 학대 글, 감염력이 필요한 작품
- **특징**: 10-15장마다 최소 1개 감정 폭발 포인트, 파도형 감정 리듬
- **트리거 키워드**: 감정 풍부, 감정 강렬, 감염력, 감동적

### 7. romance-sweet (달달한 글 규범)
- **적합**: 로맨스 달달한 글, 현대 달달 애호, 가벼운 연애 이야기
- **특징**: 달콤 밀도 설정, 갈등 조절, BE 경향 금지
- **트리거 키워드**: 달달한 글, 달달하게, 초달달, HE

### 8. romance-angst (학대 글 규범)
- **적합**: 학대 연애소설, 비극적 사랑, 칼 글
- **특징**: 학대에 이유와 절도가 있어야, 학대 포인트 설계, 학대 등급 조절
- **트리거 키워드**: 학대 글, 학대, BE, 칼, 이별 학대, 타이밍 학대

상세 키워드 목록과 트리거 시나리오는 [KEYWORDS.md](KEYWORDS.md)를 참조하세요.

## 활성화 후 워크플로

### 단계 1: 규범 선택 확인

키워드 탐지 후, 먼저 사용자 의도를 확인합니다:

```
【AI 느낌 강함, AI 느낌 제거】를 언급하신 것을 감지했습니다. 이것은 보통 anti-ai-v4 규범에 해당합니다.

다음을 할 수 있습니다:
1. 📖 이 규범의 상세 문서 보기
2. ✅ 이 규범을 프로젝트에 직접 적용
3. 🔍 다른 규범 옵션 확인

어떤 것을 원하시나요?
```

### 단계 2: 규범 지식 베이스 로드

사용자가 확인하면, Read 도구로 해당 지식 베이스 파일을 읽습니다:

```
📖 읽기: .claude/knowledge-base/requirements/{requirement-name}.md
```

핵심 요점을 표시하고, 전체 규범은 이미 컨텍스트에 로드됩니다.

### 단계 3: 설정 마커 작성

Edit 도구로 사용자의 `specification.md`에 설정을 추가합니다:

```yaml
---
writing-requirements:
  - anti-ai-v4
  - fast-paced
---
```

사용자에게 설정이 저장되었으며, /write 실행 시 자동으로 이 규범들을 따른다고 안내합니다.

## 다중 규범 처리

여러 규범 키워드가 탐지되면, 호환성 검사를 수행합니다:

- **호환 가능한 조합**: 자동 조합하고 종합 효과 설명
- **충돌하는 조합**: 모순을 알리고 해결 방안 제시

상세한 충돌 처리 로직은 [CONFLICT_RESOLUTION.md](CONFLICT_RESOLUTION.md)를 참조하세요.

## 능동적 제안

- **스타일 기반 규범 제안**: style-detector가 이미 활성화되었으면, 배합 규범을 능동적으로 추천
- **중도 조정 지원**: 사용자 피드백을 지속적으로 모니터링하며, 규범 조정을 지원

## style-detector와의 협력

본 Skill은 규범 레이어에 집중하며, style-detector(문체 레이어)와 협동합니다:

```
사용자 입력: "도시 쾌감물을 쓰려는데, 구어체로, AI 느낌도 제거해주세요"

style-detector → 식별: natural-voice
requirement-detector → 식별: fast-paced + anti-ai-v4

최종 설정:
writing-style: natural-voice
writing-requirements:
  - fast-paced
  - anti-ai-v4

효과: 자연스러운 목소리의 빠른 리듬 쾌감물, 강력한 AI 느낌 제거
```
