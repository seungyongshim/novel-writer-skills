# Novel Writer 체계적 창작 워크플로

## 사용자가 "소설을 쓰고 싶어요"라고 말하면

**즉시 이 워크플로 가이드를 활성화합니다.**

## 7단계 방법론 개요

```
1. /constitution  → 창작 원칙 수립
2. /specify       → 스토리 규격 정의
3. /clarify       → 모호한 점 명확화
4. /plan          → 창작 계획 수립
5. /tasks         → 실행 작업 분해
6. /write         → AI 보조 집필
7. /analyze       → 품질 검증 분석
```

## 상세 단계 가이드

### 1단계: 창작 원칙 수립

> "먼저 `/constitution`으로 창작 원칙을 정의합시다. 이것이 핵심 가치관, 품질 기준, 이 이야기의 양보할 수 없는 규칙을 수립합니다."

**왜 중요한가**: "창작 헌법"으로서 - 의사결정이 이 원칙을 참조합니다.

**포함 내용**:
- 핵심 가치관 (어떤 이념을 전달할 것인가)
- 품질 최저선 (절대 타협하지 않는 것)
- 스타일 원칙 (언어, 리듬, 분위기)
- 내용 원칙 (캐릭터, 플롯, 세계관 규범)

**소요 시간**: 15-20분

### 2단계: 스토리 규격 정의

> "이제 `/specify`로 제품 규격과 같은 스토리 문서를 만듭니다. 프로덕트 매니저처럼 생각하세요: 이 이야기는 무엇이고, 누구를 위해 쓰고, 무엇이 독특하게 만드나요?"

**포함 내용**:
- 한 줄 요약
- 타겟 독자
- 핵심 갈등
- 주요 캐릭터
- 성공 기준

**왜 중요한가**:
- "누구를 위해"와 "무엇을" 명확화
- 측정 가능한 성공 기준 설정
- 후속 창작에 명확한 목표 제공

**소요 시간**: 30-45분

**특수 마킹**:
- `[명확화 필요]` - 추가 명확화가 필요한 점
- `[핵심 요구]` - 타협 불가능한 요구
- `[선택 기능]` - 금상첨화 내용

### 3단계: 핵심 결정 명확화

> "`/clarify`를 실행하여 5개의 정확한 질문으로 불명확한 부분을 해결합니다. AI가 규격의 모호한 점을 식별하고, 명확한 결정을 내리도록 도와줍니다."

**왜 중요한가**: 기획의 모호함 = 집필의 혼란.

**과정**:
- AI가 규격의 모호한 점 식별
- 최대 5개 핵심 질문 생성
- 대화형 답변, 답이 바로 규격 업데이트
- 결정 이력 보존으로 되돌리기 가능

**전형적 질문**:
- "주인공의 핵심 동기는 복수인가 정의인가?"
- "이야기 리듬은 빠른 페이스인가 슬로우번인가?"
- "결말은 해피엔딩인가 여운을 남기는 건가?"

**소요 시간**: 10-15분

### 4단계: 창작 계획 수립

> "`/plan`으로 장 구조와 기술적 방법을 만듭니다. 여기서 규격을 어떻게 구현할지 결정합니다."

**설계 내용**:
- 장 분해와 활성 플롯 라인
- 리듬과 긴장감 분배
- 복선 계획
- 캐릭터 아크 매핑

**왜 중요한가**:
- 추상적 요구를 구체적 방안으로 변환
- 적합한 집필 방법과 구조 선택
- 플롯, 인물, 세계관의 구체적 구현 설계

**기술적 결정 예시**:
- "7점 구조로 서스펜스 강화"
- "다중 서사 라인으로 복잡성 전개"
- "제한 시점으로 신비감 증가"

**소요 시간**: 45-60분

### 5단계: 실행 작업 분해

> "`/tasks`를 실행하여 우선순위, 의존 관계, 예상 작업량이 포함된 실행 가능한 작업 목록을 생성합니다."

**작업 유형**:
- 장 집필 작업
- 캐릭터 프로필 보완
- 세계관 설정 보충
- 수정 및 다듬기 작업

**작업 마킹**:
- `[P]` - 병렬 실행 가능
- `[의존:X]` - 작업 X 완료에 의존
- `[고우선]` - 높은 우선순위 작업

**왜 중요한가**:
- 큰 목표를 작은 단계로, 창작 난이도 감소
- 우선순위와 의존 관계 명확화
- 병렬 창작 지원으로 효율 향상

**소요 시간**: 20-30분

### 6단계: 집필 시작

> "이제 `/write`로 장별 AI 보조 집필. 규격과 계획이 모든 장을 안내하여 일관성을 확보합니다."

**집필 시**:
- 장르 지식 스킬이 자동 활성화
- 일관성 검사기가 백그라운드에서 실행
- 리듬 모니터가 문제 감지 시 알림

**실행 원칙**:
- constitution의 원칙 엄격 준수
- plan의 기술적 방안 참조
- tasks의 구체적 작업 완료
- 규격과의 일관성 유지

**권장 리듬**:
- 매 집필 세션 1-2장
- 매 3-5장마다 멈추고 검토
- 전진 동력 유지, 과도한 편집 금지

### 7단계: 품질 검사

> "매 5장마다 `/analyze`를 실행하여 품질을 유지하고 문제를 조기 발견합니다."

**두 가지 분석 모드**:
- **프레임워크 분석** (집필 전): 기획 검증
- **내용 분석** (집필 후): 실행 검증

**검증 차원**:
- 헌법 준수 (핵심 원칙 위반 여부)
- 규격 충족도 (요구 실현 여부)
- 내용 일관성 (플롯, 시간, 인물)
- 품질 기준 (설정 기준 달성 여부)

**언제 실행하나**:
- 첫 3장 완료 후 (조기 검증)
- 매 5장 (정기 검사)
- 초고 완성 후 (전면 검토)

## Skills 자동 활성화

이 워크플로를 따를 때, 관련 스킬이 컨텍스트에 따라 활성화됩니다:

**`/specify` 중**:
- "로맨스"를 언급하면 → Romance skill 활성화
- "미스터리"를 언급하면 → Mystery skill 활성화
- "판타지"를 언급하면 → Fantasy skill 활성화

**`/write` 중**:
- 대화 집필 → Dialogue Techniques 활성화
- 장면 집필 → Scene Structure 활성화
- 모든 집필 → Consistency Checker 백그라운드 실행

**전 과정**:
- Workflow Guide (이 스킬)이 계속 활성
- 핵심 단계를 건너뛰지 않도록 확인
- 필요할 때 리마인더 제공

## 사용자가 벗어날 때

사용자가 단계를 건너뛰거나 계획 없이 바로 집필을 시작하려 하면:

**부드러운 알림**:
> "바로 집필로 넘어가시는군요. Novel-writer의 강점은 체계적인 사전 기획에 있습니다. `/constitution`과 `/specify`로 견고한 기반을 만드시겠어요? 집필 과정이 더 순조롭고 일관성 유지에 도움이 됩니다."

**사용자가 건너뛰기를 고집하면**:
> "알겠습니다 - 나중에 기획 문서를 만드셔도 됩니다. 다만 `/specify`와 `/plan` 없이는 일관성을 수동으로 추적해야 합니다."

## 프로젝트 규모별 조정

### 단편소설 (1-3만 자)

**간소화 프로세스**:
```
/specify → /clarify → /write → /analyze
```

**중점**: 핵심 갈등과 반전 명확화

**소요 시간**: 1-2일 집중 작업

### 중편소설 (10-20만 자)

**표준 프로세스**:
```
완전 7단계
```

**중점**: 계획과 작업 분해

**소요 시간**: 2-6개월

### 장편 연재 (50만 자+)

**강화 프로세스**:
```
plan → tasks → write → analyze 반복 순환
```

**중점**: 지속적 일관성 검증

**소요 시간**: 6-18개월

## 자주 묻는 질문 처리

### "이미 개요가 있어요"

> "좋습니다! `/specify`로 novel-writer 형식으로 변환합시다. 이렇게 하면 머신 리더블하게 되고 모든 추적 기능이 활성화됩니다."

**과정**:
1. 기존 개요 가져오기
2. `/specify`로 형식화
3. `/clarify`로 빈 곳 채우기
4. `/plan`으로 상세 기술 계획 생성

### "설정이 많아 보여요"

> "설정에 30-60분이 걸리지만, 수 시간의 수정을 절약합니다. 본질적으로 소설의 '소스 코드'를 만드는 것입니다 - 규격을 수정하면 내용이 재생성됩니다. 이것 없이는 변경에 전문 수동 재작성이 필요합니다."

**수익**:
- 일관성 자동 유지
- 변경사항 자동 전파
- 품질 검사 체계화
- 수정이 더 빠르고 쉽게

### "워크플로 일부만 쓸 수 있나요?"

> "네, 하지만 일부 명령은 서로 의존합니다:
> - `/write`는 `/specify`와 `/plan`이 있어야 효과적
> - `/analyze`는 평가할 작성된 장이 필요
> - `/track`은 먼저 `/track-init`이 필요
> 
> 최소 실행 가능 워크플로: `/constitution` → `/specify` → `/write`"

**완전 선택적**:
- `/clarify` - 규격이 이미 명확하면
- `/tasks` - 자유 형식 집필을 선호하면
- `/analyze` - 강력히 권장하지만

### "제 집필 스타일에 맞나요?"

**호환되는 집필 스타일**:
- **플래너**: 구조를 좋아할 것
- **팬서** (감각적 집필): 가벼운 규격 사용, `/write`에 집중
- **혼합형**: 필요에 따라 조정

**조정 제안**:
- 플래너: `/plan`에 더 많은 시간 투자
- 팬서: `/specify`를 간결하게, 발견을 허용
- 혼합형: 전체 프로세스 사용하되 유연하게

## 진도 추적

### 시각적 진도 표시

워크플로를 진행할 때, 현재 위치를 표시합니다:

```
✅ /constitution  완료
✅ /specify       완료
✅ /clarify       완료
🔄 /plan          진행 중...
⏸️ /tasks         대기
⏸️ /write         대기
⏸️ /analyze       대기
```

### 마일스톤

**기획 단계 완료** (1-5단계):
- 완전한 스토리 블루프린트 보유
- 집필 시작 준비 완료
- 모든 주요 결정 완료

**초고 완성** (6단계):
- 모든 장 집필 완료
- 전면 분석 준비
- 수정 단계 진입

**품질 검증 통과** (7단계):
- 일관성 검증 완료
- 품질 기준 충족
- 발행 또는 추가 다듬기 준비

## 다른 Skills과의 통합

### 자동 트리거

**장르를 언급하면**:
- "로맨스" → Romance skill이 관습 제공
- "미스터리" → Mystery skill이 단서 배치 제공
- "판타지" → Fantasy skill이 세계관 구축 제공

**어려움을 겪으면**:
- "캐릭터가 평면적" → Character Development 명령 제안
- "장면이 질질 끌려" → Scene Structure skill 활성화
- "대화가 딱딱해" → Dialogue Techniques skill 활성화

### 협업 워크플로

**모든 Skills이 워크플로를 인지합니다**:
- 워크플로에서 현재 위치를 참조
- 현재 단계에 적합한 제안
- 필요할 때 이전 단계 완료를 상기

## 성공 패턴

### 높은 생산성 모드

```
월요일: /constitution + /specify (2-3시간)
화요일: /clarify + /plan (2-3시간)
수요일: /tasks + /write 시작 (2-3시간)
목-일요일: /write 세션 (매일 1-2시간)
다음 월요일: /analyze + 다음 배치 계획
```

### 탐험적 모드

```
1주차: /constitution + 가벼운 /specify
2-3주차: /write 탐험적 장
4주차: 학습을 기반으로 /specify와 /plan 업데이트
5-8주차: 구조화된 /write
9주차: /analyze와 수정
```

### 스프린트 모드 (NaNoWriMo)

```
1일: 빠른 /constitution + /specify + /plan (4시간)
2-27일: 집중 /write (매일 2000자)
28-30일: /analyze + 긴급 수정
```

## 혼자가 아닙니다

**전 과정에서**:
- 안내하고 상기시켜 드립니다
- Skills이 자동으로 전문 지식 적용
- 일관성 자동 검사
- 당신은 창작에 집중, 시스템이 조직 처리

**항상 당신이 주도합니다**:
- 필요 없는 단계는 건너뛰기
- 자기 스타일에 맞게 조정
- 언제든 계획 변경
- 도구가 당신에게 봉사하지, 그 반대가 아닙니다

---

**기억하세요**: 이 워크플로는 족쇄가 아닙니다 - 발판입니다. 창작 과정을 지원하여, 강력한 것을 가능하게 합니다. 익숙해지면 제2의 천성이 되어, 이것 없이 어떻게 썼는지 궁금해질 것입니다.

**시작할 준비가 되셨나요? `/constitution`으로 시작합시다!**
