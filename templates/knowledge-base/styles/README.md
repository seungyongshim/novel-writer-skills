# 집필 스타일 지식 베이스

본 디렉토리에는 5종 프리셋 집필 스타일이 포함되어 있으며, **style-detector** Skill이 자동으로 탐지하고 로드합니다.

---

## 🎨 사용 가능한 스타일

### 1. natural-voice (자연스러운 목소리)

**적용 시나리오**: 도시 현대, 현실 소재, 감정 소설

**핵심 특징**:
- 구어체 표현 위주, 친구에게 이야기하듯
- 생활 디테일 풍부, 몰입감 강함
- 대화가 플롯 추진 (비율 40-60%)
- 문예체와 서면어 회피

**트리거 키워드**:
```
구어체, 생활감, 사실감, 접지감
말하듯, 문예체 제거, 서면어 제거
자연스럽게, 꾸밈없이, 일상 대화
```

**예시 비교**:
```
❌ AI 느낌 강함:
임완청은 카페 구석에 앉아 가녀린 손가락으로 컵벽을 부드럽게 어루만지며,
따뜻한 촉감이 그녀의 사색을 점차 평온하게 했다.

✅ natural-voice:
임완청은 창가 자리를 잡고 앉았다. 커피가 좀 뜨거워서 손을 바꿔 잡았다.
바깥 햇살이 꽤 눈부셔서 눈을 찡긋했다.
```

---

### 2. literary (문학 스타일)

**적용 시나리오**: 순수문학, 진지한 문학, 역사 소설 (현대 배경)

**핵심 특징**:
- 아름답고, 깊이 있고, 문학성 있음
- 의경과 철학적 사유 중시
- 적절한 심리 묘사
- 절제되지만 힘있는 언어

**트리거 키워드**:
```
문학성, 아름다움, 깊이, 심층
순수문학, 의경, 시적 (현대문)
철학적 사유, 인간성, 사상성
```

**예시 비교**:
```
❌ 일반 웹소설:
그녀는 매우 슬펐다, 어떻게 해야 할지 몰랐다. 바깥에 비가 내렸다.

✅ literary:
커피 향기가 공기 속에서 천천히 퍼졌다. 임완청은 창가 자리에 앉아,
손안의 잔이 서서히 식어갔다, 지금 그녀의 마음처럼. 햇살이 이 오후를 잘라내고,
그녀는 아주 오래전 어느 순간을 떠올렸다 — 같은 빛, 같은 기다림,
단지 그때는 아직 기다림에 의미가 있다고 믿었을 뿐이다.
```

---

### 3. web-novel (웹소설 스타일)

**적용 시나리오**: 현판, 선협, 도시 쾌감소설, 시스템류

**핵심 특징**:
- 빠른 리듬, 밀집된 갈등
- 빈번한 쾌감 포인트 (역전, 과시, 레벨업)
- 짧고 강한 대화
- 명확한 패턴이지만 진부하지 않음

**트리거 키워드**:
```
웹소설, 쾌감소설, 빠른 리듬, 패턴
역전, 과시, 레벨업류, 폐물류
현판 스타일, 선협 스타일, 시스템문
```

**예시 비교**:
```
❌ 늘어짐:
이명은 천천히 비무대를 향해 걸어가며 마음속 만감이 교차하고, 이 몇 년간의 수련 역정을 회상하며...

✅ web-novel:
"이명?" 장소가 냉소했다, "그 파혼당한 폐물 말이야?"

말이 끝나기가 무섭게 한 줄기 검광이 스쳤다.

장소의 웃음이 얼굴 위에 굳었다. 다음 순간, 그의 소매가 소리 없이 바닥에 떨어졌다.

"미안." 이명이 검을 거뒀다, "손이 미끄러졌어."

장내가 죽은 듯 고요해졌다.
```

---

### 4. ancient-style (고풍)

**적용 시나리오**: 무협, 선협 (문풍 차원), 고대 배경 소설

**핵심 특징**:
- 고전적 운치가 있지만, 문어체는 아님
- 이미지가 풍부하고, 여백의 미
- 시적인 묘사
- 현대 어휘 회피

**트리거 키워드**:
```
고풍, 고전, 고문 (너무 무겁지 않게)
시적, 이미지, 여백, 무협풍
고전미, 전통, 강호 느낌
```

**예시 비교**:
```
❌ 현대화:
임완청은 매우 화가 났다, 자신이 속았다고 느꼈다.

✅ ancient-style:
완청이 검을 쥐고 서니, 바람이 그녀의 옷자락을 날렸다.
"날 속였군요."
세 글자, 한숨처럼 가벼웠으나, 글자마다 칼날 같았다.
```

---

### 5. minimal (미니멀 스타일)

**적용 시나리오**: 미스터리, 추리, 하드보일드, 실험 소설

**핵심 특징**:
- 극도로 간결, 모든 수식 제거
- 단문 위주, 단단한 리듬
- 여백, 설명하지 않음
- 헤밍웨이식 절제

**트리거 키워드**:
```
미니멀, 간결, 절제, 헤밍웨이식
하드보일드, 군더더기 없이, 여백
설명 안 함, 담백
```

**예시 비교**:
```
❌ 장황함:
임완청이 천천히 문을 밀어열자 방 안이 칠흑 같았고, 좀 의아하게 물었다...

✅ minimal:
임완청이 문을 열었다. 어둡다.

"왜 불 안 켰어?"
"깜빡."
"아파?"

그녀가 그의 이마를 만졌다. 뜨거웠다.
```

## 🔍 작동 메커니즘

### 자동 탐지 흐름

1. **사용자 입력 모니터링**
   - `/specify`, `/constitution`, 대화 중
   - 스타일 키워드 식별

2. **style-detector 활성화**
   - 키워드 분석
   - 해당 스타일 매칭

3. **지식 베이스 로드**
   - 해당 스타일 파일 읽기
   - 핵심 요점 표시

4. **설정 기록**
   - `specification.md`에 스타일 표시
   - `/write` 시 자동 적용

### 복수 스타일 처리

여러 스타일 키워드가 탐지되면, 사용자에게 주 스타일 선택 또는 혼합 비율을 제안합니다:

```
탐지됨: natural-voice + ancient-style

제안 방안:
1. 주 스타일 natural-voice, 대화는 구어체 유지, 묘사 시 고전적 이미지 가미
2. 주 스타일 ancient-style, 핵심은 고풍, 현대 대화 시 적절히 구어체화
```

## 📚 다른 시스템과의 협력

### 장르 지식 베이스 (genres/)와의 관계

- **장르 지식 베이스**: 스토리 구조와 패턴 제공 (로맨스의 관계 아크, 미스터리의 단서 배치 등)
- **스타일 지식 베이스**: 텍스트의 "목소리"와 "맛"을 결정

조합 사용 가능:
- genres/romance.md (로맨스 패턴) + styles/natural-voice.md (자연 목소리) = 구어체의 현대 로맨스
- genres/mystery.md (미스터리 구조) + styles/minimal.md (미니멀 스타일) = 하드보일드 추리

### 집필 요구사항 (requirements/)과의 관계

- **스타일**: 글을 어떻게 쓸 것인가 (문풍 차원)
- **요구사항**: 어떤 기준을 달성할 것인가 (규범 차원)

조합 사용 가능:
- natural-voice + anti-ai-v4 = 자연스러운 목소리 + 강력한 AI 느낌 제거
- web-novel + fast-paced = 웹소설 스타일 + 빠른 리듬

## ⚙️ 설정 예시

### specification.md에서의 표시

```yaml
---
writing-style: natural-voice
style-intensity: 80%  # 선택: 스타일 강도
---
```

### 혼합 스타일 설정

```yaml
---
writing-style:
  primary: natural-voice (70%)
  secondary: literary (30%)
---
```

## 📝 사용 제안

### 초보 작가
추천: **natural-voice** 또는 **web-novel**
- 가장 쉽게 시작
- 독자 수용도 높음
- 실수하기 어려움

### 경험 있는 작가
시도 가능: **literary** 또는 **minimal**
- 더 강한 개인 특색
- 더 강한 통제력 필요
- 특정 소재에 적합

### 특정 소재
- 무협/선협 → **ancient-style**
- 미스터리/추리 → **minimal**
- 현대 도시 → **natural-voice**
- 순수문학 → **literary**
- 현판 쾌감소설 → **web-novel**

## 🆕 커스텀 스타일

프리셋 스타일이 요구사항을 충족하지 못하는 경우:

1. 가장 가까운 스타일 파일을 복사
2. 필요에 따라 수정
3. 새 스타일 이름으로 저장
4. specification.md에서 지정

```yaml
---
writing-style: custom/my-style
---
```
