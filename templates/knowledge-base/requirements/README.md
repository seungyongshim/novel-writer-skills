# 집필 요구사항 지식 베이스

본 디렉토리에는 8종 집필 요구사항 규범이 포함되어 있으며, **requirement-detector** Skill이 자동으로 탐지하고 로드합니다.

---

## 📋 사용 가능한 요구사항

### 1. anti-ai-v4 (AI 느낌 제거 4.0 - 급진판)

**적용 시나리오**: AI 느낌에 극도로 민감한 작가

**핵심 특징**:
- 가장 급진적인 AI 제거 방안
- 200+ 어휘 금지
- 다수 AI 문형 금지
- 일부 문학성을 희생할 수 있음

**트리거 키워드**:
```
AI 느낌 과다, 기계 느낌, AI 작법 제거
사람이 쓴 것 같이, v4.0, 급진판
사자성어 제거, 최강 AI 제거
```

---

### 2. anti-ai-v3 (AI 느낌 제거 3.1 - 표준판)

**적용 시나리오**: AI 제거와 문학성의 균형

**핵심 특징**:
- 표준 AI 제거 방안
- 필요한 서면어 보존
- 균형성 최우수

**트리거 키워드**:
```
AI 느낌 제거, 기계감 제거, v3.1, 표준판
자연스럽게, 사실감
```

---

### 3. fast-paced (쾌감소설 규범)

**적용 시나리오**: 현판, 도시 쾌감소설, 빠른 리듬 소설

**핵심 특징**:
- 챕터당 최소 2개 쾌감 포인트
- 밀집된 갈등
- 리듬 늘어짐 금지
- 챕터마다 후크

**트리거 키워드**:
```
쾌감소설, 빠른 리듬, 늘어지지 않게
챕터마다 쾌감 포인트, 역전
```

---

### 4. no-poison (독점 없는 규범)

**적용 시나리오**: 올바른 가치관, 논리 엄밀한 소설

**핵심 특징**:
- 주인공 지능 저하 회피
- 억지 오해 회피
- 성모 전개 회피
- 조연에게 합리적 동기 부여

**트리거 키워드**:
```
독점 없이, 올바른 가치관
지능 저하 금지, 억지 전개 금지, 논리적으로
```

---

### 5. serious-literature (순수문학 규범)

**적용 시나리오**: 문학성과 깊이를 추구하는 작품

**핵심 특징**:
- 사상성 중시
- 심층 인물 묘사
- 사실주의 기법
- 오락성 절제

**트리거 키워드**:
```
순수문학, 순수 문학, 깊이
사실주의, 사상성, 인간성 탐구
```

---

### 6. strong-emotion (강한 감정 규범)

**적용 시나리오**: 감정이 강렬한 소설

**핵심 특징**:
- 감정 충만
- 격렬한 갈등
- 감정 폭발 포인트 명확
- 클라이맥스 연속

**트리거 키워드**:
```
강한 감정, 감정 강렬, 감정 충만
클라이맥스 연속, 감정 폭발
```

---

### 7. romance-sweet (달달 규범)

**적용 시나리오**: 달달한 로맨스 소설

**핵심 특징**:
- 충분한 달달함
- 적절한 갈등
- 비련 회피
- HE 결말

**트리거 키워드**:
```
달달, 설탕, 달콤, 애착
비련 제거, 달달하게
```

---

### 8. romance-angst (비련 규범)

**적용 시나리오**: 비련 로맨스 소설

**핵심 특징**:
- 비련에 이유가 있어야 함
- 비련에 과정이 있어야 함
- 비련에 한도가 있어야 함
- 마지막에는 화해

**트리거 키워드**:
```
비련, 가슴 아픈, BE향
비련 원함
```

---

## 🔄 작동 메커니즘

### 자동 탐지 흐름

1. **사용자 입력 모니터링**
   - specification, constitution, 대화 중
   - 요구사항 키워드 식별

2. **requirement-detector 활성화**
   - 키워드 분석
   - 해당 요구사항 매칭

3. **지식 베이스 로드**
   - 해당 요구사항 파일 읽기
   - 핵심 규칙 표시

4. **설정 기록**
   - specification.md에 표시
   - /write 시 자동 적용

## 🎯 조합 사용

```
writing-requirements:
  - fast-paced
  - anti-ai-v4
  - no-poison
```

```

  - romance-sweet

```

```
writing-style: literary
writing-requirements:
  - romance-angst
  - strong-emotion
```

## ⚠️ 충돌 감지

일부 요구사항은 충돌하며, 시스템이 자동으로 알림합니다:

**충돌 예시 1**:
```
fast-paced (빠른 리듬) + serious-literature (순수문학)
→ 충돌: 순수문학은 느린 리듬의 복선이 필요
```

**충돌 예시 2**:
```
romance-sweet (달달) + romance-angst (비련)
→ 충돌: 달달과 비련은 대립적
```
